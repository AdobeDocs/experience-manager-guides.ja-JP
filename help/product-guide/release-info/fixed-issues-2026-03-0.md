# 2026.03.0 リリースで修正された問題

この記事では、Adobe Experience Manager Guides as a Cloud Serviceの 2026.03.0 リリースの様々な領域で修正されたバグについて説明します。

新機能と機能強化について詳しくは、[&#x200B; 2026.03.0リリースの新機能](whats-new-2026-03-0.md)を参照してください。

[2026.03.0 リリースのアップグレード手順 &#x200B;](upgrade-instructions-2026-03-0.md) について説明します。

## オーサリング

- スキーマトロン（`*.sch`）ファイルを編集して「検索と置換」機能を使用すると、「検索と置換」パネルが下部の画面から部分的に消えて表示され、入力フィールドおよびコントロールにアクセスできなくなります。 （GUIDES-38412）

## 公開

- 異なる条件付きプリセットが設定された複数のマップで同じトピックを再利用する場合、最新のマップをSalesforceに公開すると、トピックのコンテンツが上書きされ、以前に公開されたマップのユーザーに誤ったデータが表示されます。 （GUIDES-37806）
- 条件付き処理または特定のネストされたマップを含むマップのネイティブ PDFを公開すると、マップで定義された `dc:title` がPDFの表紙に表示されず、表紙のタイトルが欠落します。 （GUIDES-37733）
- マップのAEM サイト出力の生成（複合コンポーネントマッピングを使用）が失敗し、出力パスに `map_title` 変数が存在する場合にエラーが発生します。 （GUIDES-36968）
- ネイティブのPDF出力プリセットで、末尾にスラッシュを付けた出力パスを指定すると、UI によって末尾にスラッシュが自動的に追加され、特定のシナリオでPDFのアップロードが失敗する原因となった 2 つのスラッシュのパスが生成されます。 （GUIDES-34932）
- クイック公開または公開管理を使用して DITA コンテンツから生成されたAEM Sitesページを公開すると、関連する DITA アセットが意図せず公開される。 （GUIDES-27967）
- （従来のコンポーネントマッピングを使用して）AEM Sitesにマップを公開する場合、異なるマップ内のトピックのターゲットサブ要素（段落など）を `xrefs` む相互参照（`scope = peer`）は、特定の要素 ID には解決されず、代わりに親トピックにのみ解決されます。その結果、目的のセクションにスクロールするのではなく、トピックの開始時にページが読み込まれます。 （GUIDES-21802）


## 翻訳

- 特定のバージョンの言語固有のアセットとして最初に管理されていた画像（`/en/` など）が、更新されたバージョンのグローバルフォルダーに移動されてベースラインの書き出しが実行されると、新しいベースラインはその画像の古い言語固有のバージョンを引き続き参照するので、ベースラインの書き出しに失敗します。 （GUIDES-39394）
- Experience Manager Guidesの外部で作成された翻訳プロジェクトを処理すると、プロパティが見つからないという複数のエラーログメッセージ *`fmTarget`生成され* す。 （GUIDES-37804）

## ベースライン

- 動的ベースラインを作成する場合、複数の同時 API リクエストが原因でエディターが応答しなくなり、他のすべての操作が停止することがあります。 （GUIDES-39054）

## レビュー

- レビュータスクにユーザーを割り当てると、ドロップダウンに選択したプロジェクトに関連付けられているユーザーだけでなく、すべてのユーザーが一覧表示され、無効なユーザーオプションが表示されます。 （GUIDES-37781）

## レポート

- マップのレポートを開く際、フィルターパネルの読み込みに遅延が生じます（GUIDES-39385）

## 既知の問題

Adobeでは、2026.03.0 リリースの次の既知の問題を特定しました。

- 重複する ServiceNow ナレッジベースプリセットを作成すると、空白の画面が読み込まれます。 （GUIDES-42732）
- ドキュメント状態を取得する API が一部のファイルに対して null 応答を返すと、UI に誤ったドキュメント状態が表示されます。 （GUIDES-42561）
- マップダッシュボードのタブ名に基づく UI のカスタマイズは適用されません。 （GUIDES-42285）
- ファイルがエディターと検索パネルの両方で開かれている場合、エクスプローラーパネルからファイルを削除すると、そのファイルは削除され、エクスプローラーリストが更新されますが、ページを更新すると検索パネルにファイルが引き続き表示されます。 （GUIDES-41935）
- アクティブなレビュータスクを更新する際に、既にレビューの一部になっているトピックを削除してから、**更新** をクリックせずに再度追加すると、[ レビュー担当者 ] タブのレビュー担当者情報が失われます。   （GUIDES-38774）
- トピックがブックマップタグ（frontmatter、chapter、part、backmatter）または循環コンテンツの一部に含まれている場合、プレビューモードでトピックを選択しても、マップビューでそのトピックがハイライト表示されません。 （GUIDES-42416）
- Assets UI では、2 つ以上のファイルまたはフォルダーを選択すると、最初の試行で **移動** ボタンが有効になりません。 （GUIDES-42721） <br> **回避策**:2 つ以上のファイルまたはフォルダーを選択した後、数秒待ってから宛先フォルダーを選択してください。
- エディターから **ユーザーの環境設定** に移動し、エディターでプレビューモードが開いている間にルートマップを更新すると、エディターに戻った際に、マップのプレビューが空白の画面として読み込まれます。 （GUIDES-42412） <br> **回避策**：プレビューモードで使用可能な **更新** アイコンを使用してプレビューを更新すると、問題が解決します。
- 既存のテンプレートの名前を変更しても、ページが手動で更新されるまで、**出力テンプレート** パネル内の名前は更新されません。 （GUIDES-42528） <br> **回避策**：ブラウザーを更新して、更新したテンプレート名を出力テンプレートパネルに表示します。
