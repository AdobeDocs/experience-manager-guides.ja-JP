# 2024.10.0 リリースで修正された問題

この記事では、Adobe Experience Manager Guides as a Cloud Serviceの2024.10.0 リリースの様々な領域で修正されたバグについて説明します。

新機能と機能強化について詳しくは、[&#x200B; 2024.10.0リリースの新機能](whats-new-2024-10-0.md)を参照してください。

2024.10.0 リリース [&#128279;](upgrade-instructions-2024-10-0.md)の アップグレード手順について説明します。


## オーサリング

- **検索** オプションは、**Source** ビューでは機能しません。 (18610)
- `<fig>`要素がスニペットとして挿入されると、`<title>`要素が自動的に追加されます。 (18562)
- OOTB トピックの再生成または増分公開APIのエラーにより、トピックの再生成が失敗します。 (18452)
- Experience Manager Guides インスタンスが不安定になり、Assets UIにアクセスした後、エラーログのサイズが最大30 ～ 40 GB増加します。 (18414)
- `autocheckout`がxmleditorで設定されている場合、**チェックアウト**&#x200B;と&#x200B;**チェックイン**&#x200B;のアイコンが同時に表示されます。 (18088)
- Adobe Experience Manager Guides as a Cloud Serviceの2024.06.0 リリースでは、オーサービューからの画像のコピー&amp;ペーストが機能しません。（18062）
- DITA マップ内で参照されているトピックの`<conref>`が、エディター内のプレビューに反映されません。 (17794)
- 大きなDITA マップの読み込みが遅くなり、**レイアウト** ビューへの切り替えに時間がかかります。 (17590)
- DITA バージョンで、Assets UIにユーザー名が誤って表示される。 (17580)
- トピック内の重複した画像IDのエラーにより、ユーザーがトピックを保存またはオーサリングできなくなります。 (17528)
- 密なデータセットを含む多数のファイルをExperience Manager Guidesにアップロードする際に問題が発生します。（17008）
- Experience Manager Guides as a Cloud ServiceのPDF レンダリング機能で、断続的なエラーが発生します。 (16966)
- テンプレートに基づいてDITA マップを作成する際に、OOTB DXML ワークフローによってプロセスが中断され、503件のサービス不能エラーが発生します。 (16529)
- エスケープ文字で書かれた&#x200B;**特殊文字**&#x200B;は、Experience Manager Guidesにアップロードされた後、トピックから削除されます。 (16495)
- 「」によってチェックアウトされたファイルは、ファイルの編集が別のタブから移動されたとき、トークンの有効期限が切れたとき、またはユーザーがログアウトしたときに、エラーメッセージが正しく表示されません。 (15223)
- 大きなファイルはWeb エディターに読み込まれず、ブラウザーがフリーズします。 (13227)
- `<keyref>`を使用して追加された`<Topicref>`は、ネイティブ PDFに表示されません。 (11974)
- コンポーネントパス `/libs/fmdita/components/versions`はハードコードされているため、ユーザーはオーバーレイできません。 (8779)
- 新しいタブでDITA トピックまたはマップを開いて編集すると、Assets UIの選択ナビゲーションがフリーズします。 (4992)
- 大きなビデオファイルを含むDITA マップをダウンロードすると、ログでメモリ不足エラーがトリガーし、UIでエラーが発生します。 (18884)
- 「&lt;」記号を含むMathML数式を挿入すると、**無効な文字** エラーが生成されます。 (18742)
- コンテンツ取り込みプロセス中にUUIDが正しく生成されないと、取り込んだマップ内のサブマップ参照が壊れます。 (18308)
- Web エディターから作成すると、新しいDITA トピックの処理に遅延が発生する場合があります。 (16836)
- Web エディター内の&#x200B;**リポジトリ**&#x200B;内のファイルの検索に時間がかかり、結果の読み込みに失敗することがあります。 (16837)

## 公開

- ネイティブ PDF出力で、キーへの相互参照が解決されない。 (18087)
- Salesforceで既存の記事を再公開すると、**duplicate_value** エラーが断続的に発生します。 (17932)
- 顧客テンプレートのスタイル設定とコンテンツの書式設定は、レイアウトにメタデータフィールドが含まれている場合に自動的に変更され、公開されたPDFの書式設定が正しくなくなります。 (17900)
- Salesforce接続の検証がLitning URLで失敗します。 (17797)
- 参照されたPDFは、公開されたコンテンツの一括アクティベーション中に&#x200B;**一括公開ダッシュボード**&#x200B;からアクティベートされません。 (17793)
- ローカライズされたマップでは、公開されたコンテンツの一括アクティベーションは機能しません。 (17638)
- Topicmeta **オプションに追加された「** メタデータを使用」オプションを選択すると、ネイティブ PDF出力のドキュメントのプロパティにメタデータプロパティが反映されません。 (17283)
- ネイティブPDF出力の条件フィルタリングが、DITA-OTと比較して期待どおりに機能しない。 (17095)
- TOCは、ネイティブ PDF出力の`<sub>`または`<sup>` タグを尊重しません。 (17028)
- AEM サイト生成と増分パブリッシュ APIが期待どおりに機能しない。 (16666)
- ネイティブ PDFの生成が失敗し、Node.jsの依存関係の取得に関連するエラーが発生します。 (14445)
- `<Conref>`は、Web エディターの`Preview` モードとネイティブ PDF出力で解決されません。 (17827)
- キー定義を含むファイルがDITA マップと同じフォルダーにない場合、ネイティブ PDF出力のコンテンツ参照が正しく解決されません。 (15062)
- DITA マップに最大7以上の見出しが含まれている場合、レベル 7の見出しは、ネイティブ PDF出力でレベル 1の見出しとして誤って扱われます。 (19698)
- コンテンツの部分（テキスト）がエレメント内で折り返されると、テキストの前後のスペースがプレビューまたは出力フォーマットに表示されません。 (19308)
- ワークフロー内のNULL POINTER EXCEPTIONにより、手動でトリガーされたポストジェネレーションワークフローが失敗し、コンテンツが11回アップロードされます。 (18880)
- **一括公開ダッシュボード**&#x200B;では、翻訳処理中のマップに対して空白が表示されます。 (19352)


## 管理

- オーバーレイ機能のパスは、韓国語ファイル用にハードコーディングされており、正しく選択されていません。 (17089)
- Guides Extension Frameworkを使用する場合、**バージョンを保存** ダイアログに加えた変更やカスタマイズが反映されません。 (17828)
- InDesignからDITAへの変換にはハードコードされたコンフィギュレーションパスがあるため、カスタム設定ファイルは選択されません。 (16891)
- 大きな依存関係/参照を含むDITA マップをベースラインを使用してダウンロードすると、完全な参照/アセットがダウンロードされません。 (19099)


### レビュー

- レビュータスクの作成中にユーザーリストを取得する際に、ユーザー数が25を超えると失敗します。 (17329)
- タスクトピックに`<cmd>` タグが1つ以上のステップに含まれている場合、レビューパネルには、タグを含むすべてのステップの接頭辞として属性`importance`が表示されます。 (19699)

## 翻訳

- 翻訳済みアセットの参照は更新されません。 (18086)
- 複数の言語に翻訳する際に、参照が直接または間接として正しくフィルタリングされない。 (17891)
- 人間による翻訳でXLIFF プロジェクトを作成できません。 (16964)
- アクティブな翻訳プロジェクトに更新されたトピックを追加すると、トピックが重複し、プロセスが失敗します。 (7688)
- 「**構造のみを作成**」オプションを選択して作成した翻訳プロジェクトには、UUIDが割り当てられていません。 (18980)
- **翻訳ステータス**&#x200B;が&#x200B;**進行中**&#x200B;の翻訳プロジェクトを選択すると、間違ったページが開きます。 (13248)
- `<conref>`のタイトルは、Web エディターのベースラインおよび翻訳ダッシュボードで解決されません。 (16961)

## 既知の問題

- AEM Sites プリセットを開くと、トピックがダーティとしてマークされます。
- 選択したパネルは、「出力」タブからブラウザーの更新時に保持されません。
- **作成者** ビューの2つのトピック参照間でトピックをドラッグ&amp;ドロップできません。
- プリセットに適用された条件フィルタリングが、**PDFとしてダウンロード**&#x200B;経由で適用されません。
- マップパネルの単一トピックの生成では、AEM Sites プリセットで選択したすべてのトピックが生成されます（レガシー以外）。
- DITA マップの上部ツールバーから挿入すると、トピックの参照がユーザーインターフェイスで壊れて表示されます。
- DITA マップに参照がない場合、ネイティブ PDFの生成が失敗します。
- マイクロサービスが有効になっている環境で、条件を使用したAEM サイトのシングルトピックパブリッシュが失敗する。
- トピックのドキュメントの状態が&#x200B;**完了**&#x200B;に更新されると、**新しいリリースの開始** アイコンは、トピックの&#x200B;**プレビュー** モードでのみ使用できます。
