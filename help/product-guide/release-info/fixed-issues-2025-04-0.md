# 2025.04.0 リリースで修正された問題

この記事では、Adobe Experience Manager Guides as a Cloud Serviceの2025.04.0 リリースの様々な領域で修正されたバグについて説明します。

新機能と機能強化について詳しくは、[&#x200B; 2025.04.0リリースの新機能](whats-new-2025-04-0.md)を参照してください。

2025.04.0 リリース [&#128279;](upgrade-instructions-2025-04-0.md)の アップグレード手順について説明します。

## オーサリング

- エディターの特殊文字の挿入ボックスで、ドイツ語のロケールの読み込みに失敗する。 (24537)
- 新しいバージョンのDITAVAL ファイルを保存する際に追加されたコメントとラベルは、新しいバージョンのバージョン履歴に保存されません。 (24076)
- 右側のパネルの&#x200B;**ファイルプロパティ**&#x200B;の下にある送信および受信の参照は、マップファイルを切り替えると正しく更新されません。 この問題は、マップファイルに多数の参照が含まれている場合に特に発生します。 (23344)
- リポジトリーフィルターは、`ui_config.json`で定義されたカスタムフィルターの順序を保持しません。 (21193)
- `codeblock`要素で複数の行のテキストを削除すると、作成者ビューから削除できない空のスペースが作成されます。 (20672)
- **IDの自動生成** オプションが`XMLEditorConfig`で有効になっている場合でも、スニペットを使用して要素を追加したり、テンプレートを使用して作成したりすると、新しいIDが要素に対して生成されない。 (21734)
- DITA テンプレートから新しいDITA アセットを作成すると、対応するDITA テンプレートからレプリケーションプロパティがコピーされます。 (25145)
- 親フォルダー名に「&amp;」文字が含まれている場合、リポジトリーパネルからファイルプロパティにアクセスできません。 (25762)
- トピックファイルを閉じると、トピックがロックされている場合でも、新しいバージョンとして保存できます。 この問題は、`XMLEditorConfig`で「**閉じる**&#x200B;で新しいバージョンを求める」オプションが有効になっている場合に特に発生します。 (23875)
- リポジトリパネルの現在の最大幅は、コンテンツ階層が深くネストされている場合に、トピックとマップタイトルを非表示にします。 (27751)

## 公開

- 従来のAEM Sites出力では、マップのネストされたトピック内のセクションリンクが、Source モードで手動で設定した場合や、コンテンツが外部ソースから読み込まれた場合、正しく解決されません。 目的のセクションに移動する代わりに、ネストされたトピックを含むメイントピックにリダイレクトします。 (26103)
- DITA トピック内の外部リンクに`scope=external`属性が見つからない場合、特にマイクロサービスが有効になっている場合、HTML5の公開は、エラーログでこの属性が見つからないファイルを示さずに失敗します。 (25969)
- PDF プリセットで「**マルチメディアファイルを埋め込む**」オプションが有効になっている場合でも、ネイティブ PDFにビデオリンクを埋め込むことができません。 (9989)
- 新しいAEM Sites パブリッシングを使用して生成されたマッピングランディングページにメタデータプロパティを渡すことができません。 (27288)

## 管理

- トピックの作業コピーのドキュメント状態が、翻訳とベースライン UIのそのトピックのすべてのバージョンに対して表示されます。 (20674)


## レビュー

- レビューダッシュボードでレビュータスクの詳細を更新しても、更新が成功したか失敗したかは確認されません。 (8051)

## 翻訳

- Experience Manager Guidesを通じて送信された翻訳には、添付されたアセットが含まれないため、翻訳ジョブの「**開始**」ボタンがユーザーが使用できなくなります。

## 既知の問題

Adobeでは、2025.04.0 リリースの次の既知の問題を特定しました。

- ベースライン UIの参照数は、ベースラインを編集しても更新されません。 変更が保存されたときにのみ更新されます。 (28015)
- 複数のタブがエディターで開いている場合、ファイルの&#x200B;**Source** ビューで&#x200B;**取り消し**&#x200B;操作を実行すると、最後の編集が元に戻されますが、以前に開いたタブにも切り替わります。 (27891)
- **ブラウザーのスペルチェック** オプションがエディター設定で有効になっている場合でも、**AEMのスペルチェック** オプションが&#x200B;**メニュー** ドロップダウンリストに表示されます。 (27993)
- Assets UIで画像を編集して保存すると、追加のバージョンが作成され、**バージョン履歴** パネルに表示されます。 (28001)
- `codeblock`要素内の新しい行に新しいコンテンツをペーストすると、空の行が自動的に挿入されます。（27842）
- 同じベースラインを使用するプリセットを切り替えると、現在のプリセットの&#x200B;**保存** ボタンが無効になります。 (28025)
- DITA マップ内のトピックは、サブマップ内で`keydef`と`topicref`の両方として使用されている場合、AEM Sites出力で公開できません。 (22269)
- マップの複数のトピックが編集され、**すべてを閉じる** オプションを使用して閉じられ、**閉じる**&#x200B;に対するバージョンの保存を依頼する設定が有効になっている場合、アプリケーションエラーが発生します。（27931）

Adobeは、回避策に関する次の既知の問題を特定しました。

+++AEM Sites出力では、公開中にベースラインが適用されない場合、画像が壊れます。 (28043)
***回避策：***&#x200B;このようなアセットは、**Assets UI**&#x200B;から公開できます。投稿されたリンクは運用可能になります。
+++
