# 非UUIDからUUID コンテンツへの移行 {#id226TI0U20XA}


使用している現在のバージョンのExperience Manager Guidesに基づいて、UUID以外のコンテンツをUUIDに移行できます。

>[!IMPORTANT]
>
> UUID サーバーにコンテンツを移行する前に、互換性のあるAEM Guides バージョンがインストールされているUUID以外のサーバーが存在することを確認します。

## 互換性マトリックス

次のマトリックスを使用して、現在の非UUID バージョンに基づいて正しい移行パスを決定します。 これにより、移行後の移行がスムーズになります。

| 移行には非UUID バージョンが必要です | 移行後のUUID バージョン | 移行後にサポートされるアップグレードパス |
|---|---|---|
| 4.3.1非UUID | 4.3.2 UUID | バージョン 4.3.2 UUIDに移行した後、4.6.0 （UUID）を直接インストールする必要があります。 4.6.0を使用したら、バージョン 5.1.0にアップグレードし、5.1.0 Service Pack 3をインストールします。 |
| 4.6.0 サービスパック 4非UUID | 4.6.1 UUID | バージョン 4.6.1 UUIDに移行した後、直接5.1.0 （UUID）にアップグレードする必要があります。 アップグレードが完了したら、バージョン 5.1.0 Service Pack 3をインストールします。 |

## 移行時間の推定

移行ユーティリティは、アセットごとに平均50 ミリ秒の割合でアセットを処理します。 次の表では、64個のvCPU、128GB RAM、およびSSDでバックアップされたストレージで構成されたシステムの移行時間の見積もりを示します。 多くのレンディションまたは高解像度のバイナリを含む大きなリポジトリまたはアセットの場合、メモリ要件が増加する可能性があります。

>[!NOTE]
>
> 実際の移行時間は、ハードウェアパフォーマンス、ストレージスループット、AEMの同時アクティビティ、システム全体の読み込みによって異なります。


| **アセット数** | **近似。 時間** |
|-----------------|-------------------------|
| 10K | ～8～9分 |
| 50K | ～42分 |
| 10万 | ～1.4時間 |
| 25万 | ～3.5時間 |
| 50万 | ～7時間 |

| 1M | ～14時間 |
| 2M | 約28時間（約1.2日） |
