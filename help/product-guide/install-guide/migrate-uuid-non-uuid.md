# 非 UUID から UUID へのコンテンツの移行 {#id226TI0U20XA}


使用しているExperience Manager Guidesの現在のバージョンに基づいて、非 UUID のコンテンツを UUID に移行できます。

>[!IMPORTANT]
>
> コンテンツを UUID サーバーに移行する前に、UUID 以外のサーバーに互換性のあるAEM Guides バージョンがインストールされていることを確認します。

## 互換性マトリックス

次のマトリックスを使用して、現在の非 UUID のバージョンに基づいて正しい移行パスを判断します。 これにより、移行後の移行がスムーズに行われます。

| 移行には UUID 以外のバージョンが必要です | 移行後の UUID バージョン | 移行後にサポートされるアップグレードパス |
|---|---|---|
| 4.3.1 非 UUID | 4.3.2 UUID | バージョン 4.3.2 UUID への移行後、4.6.0 （UUID）を直接インストールする必要があります。 4.6.0 を使用したら、バージョン 5.1.0 にアップグレードしてから、5.1.0 サービスパック 3 をインストールします。 |
| 4.6.0 サービスパック 4 非 UUID | 4.6.1 UUID | バージョン 4.6.1 UUID に移行した後、5.1.0 （UUID）に直接アップグレードする必要があります。 アップグレードが完了したら、バージョン 5.1.0 サービスパック 3 をインストールします。 |

## 移行時間の見積もり

移行ユーティリティは、アセットあたり平均 50 ミリ秒以内でアセットを処理します。 次の表に、64 vCPU、128 GB RAM および SSD ベースのストレージで構成されたシステムの移行時間の予測値を示します。 多数のレンディションや高解像度のバイナリを含む大規模なリポジトリやアセットでは、メモリ要件が増える場合があります。

>[!NOTE]
>
> 実際の移行時間は、ハードウェアのパフォーマンス、ストレージのスループット、AEMの同時アクティビティ、システム全体の負荷によって異なる場合があります。


| **アセット数** | **約 時間** |
|-----------------|-------------------------|
| 10K | ～8～9 分 |
| 50K | 最大 42 分 |
| 10 万 | ～1.4 時間 |
| 250K | ～3.5 時間 |
| 50 万 | ～7 時間 |
| 750K | ～10.5 時間 |
| 1M | ～14 時間 |
| 2M | ～28 時間（～1.2 日） |
| 3M | ～42 時間（～1.75 日） |
| 5M | ～69 時間（～2.9 日） |
| 10M | ～139 時間（～5.8 日） |
