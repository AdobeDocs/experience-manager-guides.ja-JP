# Web エディターの機能 {#id176NC500V5Z}

この節では、Web エディターで使用できるさまざまな機能について説明します。 Web エディターは、次のセクションまたは領域に分割できます。

- [メインツールバー](#id2051EA0G05Z)
- [セカンダリツールバー](#id2051EA0J0Y4)
- [左パネル](#id2051EA0M0HS)
- [コンテンツ編集エリア](#id2051EB000UI)
- [右側のパネル](#id2051EB003YK)

次のサブセクションでは、Web エディターの様々なセクションについて詳しく説明します。

## メインツールバー {#id2051EA0G05Z}

メインツールバーはWeb エディターのインターフェイスの上部にあり、Web エディターで利用できるファイルレベルの機能と様々なオーサリングモードを提供します。 上部のツールバーで使用できる機能は、次のように説明されます。

**すべて保存** - ![](images/SaveFloppy_icon.svg)

開いているすべてのトピックで行った変更を保存します。 Web エディターで複数のトピックを開いている場合は、**すべて保存**&#x200B;をクリックするか、**Crtl**+**S** ショートカットキーを使用すると、すべてのドキュメントをワンクリックで保存できます。 各ドキュメントを個別に保存する必要はありません。

>[!NOTE]
>
> 保存操作では、トピックの新しいバージョンは作成されません。 新しいバージョンを作成するには、「新しいバージョンとして保存」を選択します。

**新しいバージョンとして保存** - ![](images/save-revision-icon.png)

トピックで行った変更を保存し、トピックの新しいバージョンを作成します。 新しく作成したトピックに取り組んでいる場合、バージョン情報は&#x200B;**none**&#x200B;と表示されます。

![](images/save-all-first-version-none_cs.png){width="800"}

バージョン番号は、トピックファイルまたはマップファイル用に作成された新しいバージョンごとに変更されます。

**新しいバージョンとして保存**&#x200B;を使用してトピックまたはマップを保存することを選択すると、次のダイアログが表示されます。

![](images/save-as-new-version-dialog.PNG){width="300"}

コメントとバージョンラベルを入力して変更を特定し、**保存**&#x200B;をクリックして、ファイルの新しいバージョンを作成します。

*新しいバージョンとして保存*&#x200B;を選択すると、トピックの最初のバージョンがDAMで作成され、これはトピックの現在アクティブなバージョンにもなります。 後で、古いバージョンのトピックに戻すと、それが現在アクティブなバージョンのトピックになります。

管理者がバージョンラベルを事前に設定している場合は、ドロップダウンリストにそのラベルが表示されます。 使用可能なラベルのリストからラベルを選択し、ドキュメントを保存できます。

![](images/web-editor-pre-defined-labels.PNG){width="300"}

トピックの保存時に、トピックで行った変更を指定してコメントを追加できます。 このコメントは、トピックのバージョン履歴に表示されます。

トピックがレビュー中の場合、レビュー担当者には、新しいバージョンのトピックが利用可能であることを知らせる通知が届きます。 ドキュメントの最新リビジョンに簡単にアクセスして、トピックの最新バージョンを引き続き確認できます。

トピックのタイトルにポインターを合わせると、ファイルパスとバージョン番号が表示されます。

![](images/mouse-hover-on-title_cs.png){width="800"}

>[!NOTE]
>
> トピックのバージョンが利用可能になると、トピックにラベルを追加することもできます。 これらのラベルを使用して、ドキュメントの特定のバージョンを公開するためのベースラインを作成できます。 トピックでのラベルの使用について詳しくは、[&#x200B; ラベルの使用](web-editor-use-label.md#)を参照してください。

**取り消しとやり直し** - ![](images/Undo_icon.svg) / ![](images/Redo_icon.svg)

最後のアクションの取り消しまたはやり直し。

**要素を削除** - ![](images/Delete_icon.svg)

現在選択されているエレメントまたはカーソルが配置されているエレメントを削除します。

**検索と置換** - ![](images/FindAndReplace_icon.svg)

検索と置換機能は、作成者モードとSource ビューモードで使用できます。 「検索と置換」テキストバーがトピック編集領域の下部に表示されます。 ショートカットキー&#x200B;**CTRL**+**F**&#x200B;を使用して、検索と置換バーを呼び出すことができます。

![](images/find-replace-bar.png){width="800"}

設定アイコン \（![](images/settings-find-replace-icon.svg)\）を使用すると、**大文字と小文字を区別**&#x200B;および&#x200B;**文字全体のみ**&#x200B;検索オプションを切り替えることができます。 大文字と小文字を区別しない検索を実行するには、「**大文字と小文字を区別しない**」オプションを\（または選択\）します。 大文字と小文字を区別して検索する場合は、「**大文字と小文字を区別しない**」オプションを「\（または選択解除\）」オフにします。 また、単語全体を検索することもできます。

検索は瞬時に行われます。つまり、**検索** フィールドに検索語句または単語を入力すると、その語句は直ちに検索され、トピック内で選択されます。 同様に、トピック内のテキストを置き換えるには、検索キーワードと置き換えフィールドを入力し、**置換**&#x200B;または&#x200B;**すべてを置換** ボタンをクリックします。

Source ビューでは、「検索と置換」は、特定のエレメントや属性を検索するのに非常に便利です。 例えば、`@product`属性の値を置き換える場合は、Source ビューから簡単に行えます。 作成者ビューでは、属性または要素に基づいて検索することはできません。 ただし、XML コードが上書きされる可能性があるため、**すべて置換**&#x200B;機能の使用中は注意が必要です。

**エディター設定** - ![](images/editor_settings_icon.svg)

エディター設定は、管理ユーザーのみが使用できます。 管理者は、環境設定を使用して、次の設定を行うことができます。

>[!NOTE]
>
> デフォルト設定を更新する場合は、変更を有効にするためにドキュメントを再度開く必要があります。

- **一般**：一般設定では、Web エディターで使用する辞書を設定できます。 このタブには、**スペルチェック**、**条件**、**オーサリング**&#x200B;の3つのセクションがあります。

  ![](images/editor-setting-general.png){width="650"}

   - **スペルチェック**: **AEM スペルチェック**&#x200B;と&#x200B;**ブラウザーのスペルチェック**&#x200B;の2つのオプションがあります。 デフォルトでは、エディターはブラウザーのスペルチェック機能を使用します。スペルチェックは、ブラウザーのビルド内辞書を使用して実行されます。 AEM スペルチェックに切り替えて、AEMの辞書を使用できます。また、カスタム単語リストを追加するようにカスタマイズすることもできます。 AEM ディクショナリのカスタマイズについて詳しくは、「AEM as a Cloud Serviceのインストールと設定」の「*Adobe Experience Manager Guidesのデフォルトディクショナリのカスタマイズ*」セクションを参照してください。


   - **条件**

      - **作成者ビューでコンディショナルテキストを強調表示**：作成者ビューでコンディショナルテキストを強調表示するには、これを選択します。 条件コンテンツは、条件に定義された色を使用してハイライト表示されます。

      - **条件属性で検証**：属性に対して定義された値の検証を許可するには、これを選択します。 これにより、誤った値を追加しないようにします。

      - **件名パネルにタイトル付きのキーを表示**：これを選択すると、件名スキームのタイトルと共にキーが表示されます。 このオプションを選択しない場合は、タイトルのみが表示されます。 例えば、ここではキー「os」、「audience」、「other」もタイトルと共に表示されます。

        ![](images/subject-scheme-title.png){width="550"}

      - **条件パネルで件名スキームを表示**：条件パネルで件名スキームを表示するには、これを選択します。 これを選択解除すると、定義された条件が条件パネルに表示されます。

   - **オーサリング**

      - **すべてを置換を有効にする**：これを選択すると、検索と置換パネルに「すべてを置換」アイコンが表示されます。


   - **引用**
引用のスタイルを変更します。 プロジェクトで使用するドロップダウンから引用スタイルを選択します。 詳細については、[引用スタイルの変更](./web-editor-apply-citations.md#change-citation-style)を参照してください。


**パネル**：この設定は、エディターの左側のパネルに表示されるパネルを制御します。 目的のパネルの表示と非表示を切り替えることができます。

![](images/editor-setting-panel.png){width="650"}

>[!NOTE]
>
> カスタムパネルが設定されている場合、パネルのリストにも表示されます。 カスタムパネルの表示と非表示を切り替えることができます。 設定について詳しくは、「Adobe Experience Manager Guides as a Cloud Serviceのインストールと設定」の左側のパネル *の「* カスタムパネルの設定」セクションを参照してください。

- **要素リスト**：作成者が[要素を挿入](#id204SG30105Z) ポップアップを使用して挿入できる要素のリストを管理者が制御でき、要素の表示名も定義できます。 エレメントのリスト設定では、DITAの仕様に従ってエレメントの名前を指定し、DITAで定義されたエレメント名の代わりに使用するラベルを指定できます。

  ![](images/editor-setting-element-list.png){width="650"}

上のスクリーンショットでは、`b`要素に太字のラベルが付けられ、`codeblock`には他の要素と共にコードブロックのラベルが付けられます。 「**上記のエレメントのみを使用**」オプションを選択すると、このリストの有効なエレメント \（現在の挿入ポイントで\）のみがエレメントを挿入ポップアップに表示されます。

次のスクリーンショットでは、前のスクリーンショットで設定した4つの要素のうち、現在のコンテキストに表示されているのは3つだけです。

![](images/editor-setting-insert-element-list.PNG){width="300"}

- **属性リスト**：要素リストと同様に、要素の属性リストに表示する属性とその表示名のリストを制御できます。 次のスクリーンショットでは、要素の属性リストに表示するように設定されている属性は3つだけです。

  ![](images/editor-setting-attributes-list.png){width="650"}

  この設定では、要素に属性を追加しようとすると、リストに設定された属性のリストのみが表示されます。

  ![](images/editor-setting-add-attributes-list.png-to-element.PNG){width="300"}

- **公開プロファイル**：これには、**ナレッジベース**&#x200B;出力の公開に使用できる公開プロファイルが含まれます。 ターゲット ナレッジベース用の新しいプロファイルを作成できます。 例えば、SalesforceやServiceNowなどです。

   - **Salesforce公開プロファイルの作成**

     **前提条件**

      - Salesforce用に接続アプリを作成します。 詳しくは、[API統合のOAuth設定を有効にする](https://help.salesforce.com/s/articleView?id=sf.connected_app_create_api_integration.htm&type=5)を参照してください。

      - 接続されたアプリを設定する際は、次のことを確認します。

         - コールバックを指定します。

           `URL: http://<server name>:<port>/bin/dxml/thirdparty/callback/salesforce`

         - 次のOAuth スコープを選択します。
            - フルアクセス （フル）
            - 「APIを介してユーザーデータを管理」（API）を選択します

  アプリが設定されると、Salesforceは&#x200B;**Consumer Key**&#x200B;と&#x200B;**Consumer Secret**&#x200B;を提供します。

  これらを使用して、Salesforce公開プロファイルを作成できます。


   - Salesforce公開プロファイルを作成するには、**Server Type** ドロップダウンから&#x200B;**Salesforce** Knowledge Baseを選択します。 プロファイル名を入力します。 **サイト URL**&#x200B;に、出力の公開に使用するコンシューマーサイトを入力し、Salesforce コンシューマーサイトから提供された&#x200B;**Consumer Key**&#x200B;および&#x200B;**Consumer Secret**&#x200B;を追加します。 次に、**新しく作成したプロファイルを**&#x200B;検証し、**保存**&#x200B;します。
     ![&#x200B; エディター設定のsalesforce パブリッシュプロファイル &#x200B;](./images/salesforce-publish-profile.png){width="550"}

     >[!NOTE]
     >
     >AEMでSalesforceのプロキシを設定するには、Experience Manager GuidesでApache HTTP Components Proxy Configurationを使用します。 AEM Link Checker[&#128279;](https://helpx.adobe.com/experience-manager/kb/How-to-configure-proxy-for-the-AEM-Link-Checker-AEM.html)のプロキシを設定する方法について説明します。


   - **ServiceNow公開プロファイルの作成**

     **前提条件**

     アセットをアップロードするようにServiceNow サーバーを設定します。
      - **ServiceNow** サーバーに接続します。
      - **システムのプロパティ** > **セキュリティ**&#x200B;に移動します。
      - 次のオプションのチェックを外します。

        **このプロパティは、アップロードのMIME タイプのチェックをアクティブ化するように設定する必要があります（すべてのバージョン Eureka以降）。 添付ファイルのMIME タイプ検証を有効（true）または無効（false）にします。 glide.attachment.extensionsで設定されたファイル拡張子は、アップロード時にMIME タイプがチェックされます。**

      - 「**保存**」をクリックします。

     アプリを設定したら、**ServiceNow**&#x200B;公開プロファイルを作成します。
   - 公開プロファイルを作成するには、**サーバータイプ** ドロップダウンからServiceNow ナレッジベースを選択します。 プロファイル **名前**&#x200B;を入力します。 **ServiceNow URL**&#x200B;に、出力の公開に使用するコンシューマーサイトを入力し、ServiceNow コンシューマーサイトから提供される&#x200B;**ユーザー名**&#x200B;と&#x200B;**パスワード**&#x200B;を追加します。 次に、**新しく作成したプロファイルを**&#x200B;検証し、**保存**&#x200B;します。

     ![ServiceNow パブリッシュプロファイル &#x200B;](./images/service-now-publish-profile.png){width="550"}

  検証後、DITA マップの出力プリセットで公開プロファイルを選択し、それを使用して、選択した&#x200B;**Salesforce**&#x200B;または&#x200B;**ServiceNow** サーバーに出力を生成できます。

  [&#x200B; ナレッジベース &#x200B;](../user-guide/generate-output-knowledge-base.md)出力プリセットについて詳しく説明します。


- **検証**：このタブには、Web エディターでSchematron検証を設定するオプションが含まれています。 次の機能を有効にできます。

   - **ファイルを保存する前に検証チェックを実行する**：選択したSchematron ファイルを使用してSchematron検証を実行するには、保存操作の前に、これを選択します。 「+」アイコンをクリックして、Schematron ファイルを追加できます。 選択したSchematron ファイルが一覧表示されます。

     >[!NOTE]
     >選択したSchematron ファイルは、選択したフォルダープロファイルに対して保持されます。

     ![&#x200B; エディター設定での検証](./images/editor-setting-validation.png){width="550"}
これにより、選択したSchematron ファイルで定義されたルールを破るファイルをユーザーが保存できなくなります。 これが選択されていない場合、変更を保存する前にファイルが検証されません。

   - **すべてのユーザーが検証パネルにスキーマトロンファイルを追加することを許可する**：これを選択すると、ユーザーはWeb エディターの検証パネルにスキーマトロンファイルを追加できます。 これにより、ユーザーはSchematron ファイルを追加し、Schematron ファイルに対してトピックを検証できます。 これが選択されていない場合、**スキーマトロンファイルを追加** ボタンは、Web エディターの&#x200B;**検証パネル**&#x200B;のユーザーには使用できません。


- **表示属性**：属性リストと同様に、要素の属性リストに表示する属性のリストを制御できます。 デフォルトでは、4つの&#x200B;**表示属性** — オーディエンス、プラットフォーム、製品、およびpropが、要素の属性リストに表示されるように設定されています。 上部の&#x200B;**追加** アイコンを使用して、表示属性を追加することもできます。 **削除** アイコンを使用して、表示属性のいずれかを削除することもできます。

  エレメントに対して定義された属性は、レイアウトおよびアウトライン表示に表示されます。

  ![](images/editor-settings-display-attributes.png){width="550"}

- **翻訳**：このタブには、言語グループの作成、ソースラベルのターゲットバージョンへの反映、翻訳プロジェクトのクリーンアップを行うためのオプションが含まれています。
  ![](images/editor-setting-translation.png){width="550"}

   - **言語グループ**：管理者は、言語のグループを作成し、それらをセットとして使用してコンテンツを翻訳できます。\
     新しい言語グループを作成するには、次の手順を実行します。
      1. 「![追加アイコン &#x200B;](images/Add_icon.svg)を追加」アイコンを選択します。
      1. 言語グループ名を入力します。 各言語には一意の名前を付ける必要があります。 名前フィールドが空の場合、または名前が一意でない場合は、エラーを表示できます。
      1. ドロップダウンから言語を選択します。 複数の言語を選択できます。

     言語の最初の数文字、または言語コードを入力して、目的の言語をフィルタリングします。 例えば、名前またはコードの先頭に「en」が含まれているすべての言語をフィルタリングするには、「en」と入力します。
      1. 選択した言語をグループに追加するには、**完了**&#x200B;を選択します。 言語が表示されます。 3つ以上の言語を追加すると、**さらに表示**&#x200B;が表示されます。 **詳細を表示**&#x200B;を選択して、グループに存在するすべての言語を表示できます。

         >[!TIP]
         >
         > **表示の切り替え**&#x200B;から&#x200B;**表示の切り替え**&#x200B;の切り替え。一部の言語のみを表示します。

      1. グループ内の言語にカーソルを合わせて![編集アイコン &#x200B;](images/edit_pencil_icon.svg)を編集するか、言語グループを![削除](images/Delete_icon.svg)します。
      1. **エディター設定**&#x200B;を保存します。

         >[!NOTE]
         >
         >ユーザーは、フォルダープロファイルに設定された言語グループを表示できます。

   - **ソースバージョンラベルをターゲットバージョンに反映**：このオプションを選択すると、ソースファイルバージョンのラベルが翻訳ファイルに渡されます。 デフォルトでは、これは無効になっています。
   - **完了後の翻訳プロジェクトのクリーンアップ**：翻訳後に翻訳プロジェクトを自動的に無効または削除するように設定するには、このオプションを選択します。 デフォルトでは、**なし**&#x200B;が選択されているため、翻訳後にプロジェクトが存在します。

     翻訳プロジェクトを後で使用する場合は、翻訳プロジェクトを無効にできます。 プロジェクトを削除すると、プロジェクトに存在するすべてのファイルとフォルダーが完全に削除されます。


- **メタデータ**: トピックのバージョンメタデータと、その値を&#x200B;**バージョン履歴** ダイアログボックスに表示するように制御できます。  メタデータパスで、メタデータを選択するノードの場所を指定します。 メタデータのカスタム名をラベルとして定義することもできます。 デフォルトのプロパティは、タイトル、ドキュメント状態、タグです。

  メタデータは、アセットの`/jcr:content` ノードの下にある任意のプロパティから選択できるため、プロパティのパスをメタデータパスとして追加できます。

## セカンダリツールバー {#id2051EA0J0Y4}

Web エディターで編集用のトピックを開くと、セカンダリツールバーが表示されます。 セカンダリツールバーで使用できる機能は、次のように説明されます。

**エレメントを挿入** - ![](images/Add_icon.svg)

現在または次の有効な場所に有効な要素を挿入します。 キーボードショートカット ***Alt***+***Enter***&#x200B;を使用して、エレメントの挿入ポップアップを開くこともできます。 例えば、段落を編集している場合、**エレメントを挿入** ポップアップに、段落に挿入できるエレメントのリストが表示されます。 挿入するエレメントを選択します。 キーボードを使用してエレメントのリストをスクロールし、***Enter***&#x200B;を押して必要なエレメントを挿入できます。

有効なエレメントは、次の2種類があります。

- **現在の場所の有効な要素**：現在のカーソルの場所に挿入できる要素がリストに表示されます。

- **現在の場所の外部にある有効な要素**：このリストには、エレメント階層内の現在の要素に対する親のいずれかの後に挿入できる要素が表示されます。



例えば、インライン `<b>`要素内にいる場合は、現在の場所に`<u>`、`<xref>`、`<i>`などの要素を挿入できます。 対照的に、`<table>`や`<topic>`などの要素を現在の場所の外に挿入することもできます。

検索ボックスに文字または文字列を入力して、その文字で始まる要素を検索することもできます。


![&#x200B; エレメントを挿入](images/insert-element.png){width="300"}

*tと入力して、&#39;t&#39;で始まるすべての有効な要素を検索します。*

`note`などのブロック要素の内部で作業している場合は、「要素を挿入」アイコンを使用して、`note`要素の後に新しい要素を挿入します。 次のスクリーンショットでは、p \（paragraph\）要素の中にメモ要素が挿入されています。

![&#x200B; ブロック要素に要素を挿入](images/note-in-para-insert-element_cs.png){width="800"}

ノート要素でEnter キーを押すと、ノート要素自体に新しい段落が作成されます。 ノートの外側に新しいエレメントを挿入するには、エレメントのパンくずリストでp エレメント \（スクリーンショットで強調表示\）をクリックし、エレメントを挿入アイコンをクリックするか、***Alt***+***Enter***&#x200B;を押してエレメントを挿入ポップアップを開きます。 次に、目的のエレメントを選択し、Enter キーを押して、選択したエレメントをノートエレメントの後に挿入します。

点滅するブロックカーソルが表示されたときに、2つのエレメント間にエレメントを追加することもできます。

![](images/Block-cursor.png){width="300"}

例えば、DITA トピックを扱っていて、ブロックカーソルが短い説明と本文の間で点滅している場合、`prolog`要素を追加してから、著作権、作成者、その他の詳細を追加できます。

新しいエレメントを入力するもうひとつの方法は、コンテキストメニューを使用することです。 文書内の任意の場所を右クリックして、コンテキストメニューを呼び出します。 このメニューから、**エレメントを挿入**&#x200B;を選択して、**エレメントを挿入** ダイアログボックスを表示し、挿入するエレメントを選択します。

![](images/insert-element-before-after.png){width="300"}

**段落を挿入** - ![](images/Paragraph_icon.svg)

現在または次の有効な場所に段落エレメントを挿入します。

**番号付きリストの挿入/削除** - ![](images/TextNumbered_icon.svg)

現在または次の有効な場所に番号付きリストを作成します。 番号付きリストを使用してこのアイコンをクリックすると、その項目は通常の段落に変換されます。

**箇条書きリストの挿入/削除** - ![](images/BulletList_icon.svg)

現在または次の有効な場所に箇条書きリストを作成します。 箇条書きリストを開き、このアイコンをクリックすると、その項目は通常の段落に変換されます。

>[!NOTE]

>リスト項目のコンテキストメニューから「**リストを分割**」オプションを選択して、現在のリストを分割し、同じレベルで新しいリストを開始することもできます。

**テーブルを挿入** - ![](images/Table_icon.svg)

現在または次の有効な場所にテーブルを挿入します。 表を挿入アイコンをクリックして、表を挿入ダイアログを開きます。

![](images/table-properties.png){width="550"}

テーブルに必要な行と列の数を指定できます。 最初の行を表ヘッダーとして保持する場合は、「最初の行をヘッダーとして設定」オプションを選択します。 表にタイトルを追加するには、「タイトル」フィールドにタイトルを入力します。

表を挿入したら、コンテキストメニューを使用して表を修正できます。

![](images/table-context-menu_cs.png){width="550"}

テーブルのコンテキストメニューを使用すると、次のことが可能になります。

- セル、行または列の挿入

- セルを右方向と下方向に結合

- セルを水平方向または垂直方向に分割

- セル、行または列の削除

- テーブルからスニペットを作成する

- IDの生成


また、複数のセル、行全体、またはテーブルの列に対して属性を定義することもできます。 例えば、表のセルを整列させるには、目的のセルをドラッグして選択します。 コンテンツのプロパティパネル（右側）で、プロパティ **Type**&#x200B;が&#x200B;**複数エントリ**&#x200B;に変更されます。

1. **属性** セクションで、**+追加**&#x200B;をクリックします。
1. **属性** ドロップダウンリストから`@valign`属性を選択します。
1. 値ドロップダウンリストから、選択した表セルに適用するテキストの整列を選択します。
1. 「**追加」をクリックします。**

![](images/align-table-cell_cs.png){width="800"}

**画像を挿入** - ![](images/Image_icon.svg)

現在または次の有効な場所に画像を挿入します。 画像を挿入アイコンをクリックして画像を挿入ダイアログを開き、挿入する画像を検索して選択します。

>[!NOTE]
>
> ローカルシステムから記事に画像をドラッグ&amp;ドロップして、画像を追加することもできます。 この場合、画像ファイルは&#x200B;**Assetsをアップロード** ワークフローを使用して追加されます。  詳しくは、[左パネル &#x200B;](web-editor-features.md#id2051EA0M0HS) セクションの「**Assets**&#x200B;をアップロード」ワークフローを参照してください。


![](images/insert-image.png){width="650"}

画像の挿入ダイアログで、画像/図のタイトルと代替テキストを画像に追加できます。

上部の検索バーにファイル名を入力して、必要な画像ファイルを検索できます。また、パス \（検索バー\）、コレクション、ファイルタイプ、タグで検索結果をフィルタリングすることもできます。 必要な画像ファイルが見つかったら、ファイルを選択し、「選択」をクリックして文書に画像を挿入します。 `.png`、`.svg`、`.gif`、`.jpg`、`.eps`、`.ai`、`.psd`など、様々な形式の画像ファイルを挿入できます。

画像を挿入したら、コンテンツプロパティパネルから高さ、幅、配置、属性を変更できます。 画像ファイルをクリックし、右側のパネルのコンテンツプロパティパネルで変更を行います。

![](images/image-properties.png){width="800"}

Source フィールドには、挿入された画像ファイルのUUIDが表示されます。 Source フィールドの上にマウスポインターを置くと、挿入された画像ファイルのパス全体を確認できます。 パスがツールチップに表示されます。

画像ファイルに「高さ」または「幅」の値を指定することで、画像のサイズを変更できます。 画像の縦横比は自動的に維持されます。 必要に応じて、ロックアイコン\（縦横比を維持\）をクリックし、「高さ」と「幅」の値を指定して、画像ファイルの縦横比を維持しないように選択することもできます。

画像の配置設定をインラインまたはブレークとして指定することもできます。 「配置を分割」オプションを使用する場合は、画像を整列させる場所（左、中央、または右）を選択できます。

**属性** フィールドで必要なプロパティを選択して、画像ファイルの他のプロパティを追加することもできます。

>[!NOTE]
>
>画像内のクリック可能な領域\（画像マップ\）を定義することもできます。 詳しくは、[左パネル &#x200B;](web-editor-features.md#id2051EA0M0HS) セクションの&#x200B;**画像マップの挿入/編集**&#x200B;機能の説明を参照してください。

**画像またはメディアファイルのコンテキストメニュー**

また、コンテキストメニューを使用して、画像やメディアファイルに対して一般的な操作を実行することもできます。 画像上の任意の場所を右クリックして、コンテキストメニューを呼び出します。

コンテキストメニューには、画像またはメディアをカット、コピー、またはペーストするオプションが用意されています。 選択したエレメントの前または後にエレメントを挿入できます。 また、エレメントの名前を変更したり、ラップを解除したりするオプションもあります。 選択した画像またはメディアをリポジトリ内で見つけるか、Assets UIでファイルのプレビューを表示できます。

コンテキストメニューの他のオプションでは、パスのコピー、画像マップの編集、スニペットの作成、選択した要素のIDの生成を行うことができます。

**マルチメディアの挿入** - ![](images/insert-multimedia-icon.svg)

様々なタイプのマルチメディアファイルを挿入します。 マルチメディアを挿入アイコンをクリックし、挿入するファイルのタイプを選択します。 サポートされているマルチメディア形式は次のとおりです。

- オーディオファイル
- ビデオファイル
- YouTube
- ヴィメオ

「オーディオ」または「ビデオファイル」オプションを選択すると、目的のファイルを参照して選択するためのリポジトリビューが表示されます。 YouTubeまたはVimeoを選択すると、挿入マルチメディア ダイアログが表示されます。 「Web リンク」フィールドにビデオファイルのリンクを貼り付け、「挿入」をクリックして、ドキュメント内の現在または次の有効な場所にビデオを追加します。

>[!NOTE]
>
> YouTube ビデオリンクの追加中に、URLの文字列`watch?v=`を`embed`に置き換える必要があります。 例えば、次のYouTube ビデオリンクを追加するには、次のように追加する必要があります：`https://www.youtube.com/**embed/**WlIKQOrmZcs`。 `https://www.youtube.com/**watch?v**=WlIKQOrmZcs`この変更により、ビデオがAEM サイトとPDF出力に確実に埋め込まれます。

マルチメディアを挿入ダイアログからオーディオまたはビデオファイルを追加することもできます。 「オーディオ/ビデオファイル」オプションを選択し、参照アイコンをクリックしてリポジトリビューを起動します。 リポジトリからオーディオまたはビデオファイルを選択し、「選択」をクリックして、オーディオ/ビデオファイル フィールドにファイルのリンクを追加します。 ビデオファイルを選択した場合、ファイルのプレビューもプレビュー領域に表示されます。 ビデオファイルを再生して、プレビューを確認できます。

![](images/insert-multimedia.png){width="650"}

**相互参照を挿入** - ![](images/Reference_icon.svg)

タイプの参照（コンテンツ参照、コンテンツキー参照、キー参照、ファイル参照、Web リンク、メールリンク）を挿入します。

**ファイルを選択** アイコン \（コンテンツ参照とファイル参照\）または&#x200B;**マップ** アイコン \（コンテンツキー参照とキー参照\）を選択し、リンク先のファイルまたはコンテンツを選択します。

![](images/insert-references.png){width="650"}

選択した参照のリンクが文書に追加されます。 リンクのコンテキストメニューでは、次のオプションを使用できます。

- **エレメントを挿入**：指定されたコンテキストで挿入できる有効なエレメントのリストを表示します。
- **UUIDをコピー**：挿入された参照のUUIDをコピーします。
- **パスをコピー**：挿入された参照の完全なパスをコピーします。
- **スニペットを作成**：挿入した参照から再利用可能なスニペットを作成します。
- **Generate ID**：挿入された参照の一意のIDを生成します。

参照するファイルのUUIDを使用して検索することもできます。 コンテンツとキー参照リンクの場合は、リンク先のファイルのUUIDを入力すると、ファイルが自動的に検索され、「プレビュー」セクションに表示されます。 ファイルのUUIDを指定する場合、.xml ファイルのファイル拡張子を明示的に指定する必要はありません。 .xml拡張子は、UUIDに自動的に追加されます。

![](images/insert-content-using-uuid-search.png){width="650"}

管理者が&#x200B;*XMLEditorConfig*&#x200B;でUUID オプションを有効にしている場合、**Link** プロパティに参照コンテンツのUUIDが表示されます。

![](images/ref-link-uuid_cs.png){width="800"}

>[!NOTE]
>

**チェックアウト/チェックイン** - ![](images/LockClosed_icon.svg)/ ![](images/LockOpen_icon.svg)

現在のファイルをチェックアウトまたはチェックインします。 ファイルをチェックアウトすると、そのファイルに対する排他的な書き込みアクセス権がユーザーに付与されます。 ファイルがチェックインされると、変更内容は現在のバージョンのファイルに保存されます。

マップビューで親マップを展開している場合は、マップ内のすべてのファイルをワンクリックでチェックアウトできます。 親マップファイルを展開して親ファイルを選択するだけで、マップ内のすべてのファイルが選択されます。 次に、**チェックアウト** ![](images/LockClosed_icon.svg)を選択して、マップ内のすべてのファイルのロックを取得できます。

>[!NOTE]
>
> 保存されていない変更があるファイルをチェックインすると、変更を保存するよう求められます。 変更を保存しない場合は、ファイル内でのみチェックされます。

チェックイン/チェックアウトのツールヒントは、`ui_config.json` ファイルのtitle プロパティによって決まります。

詳しくは、オンプレミス インストールおよび設定ガイドの「[&#x200B; チェックインおよびチェックアウト アイコンのタイトルを設定](/help/product-guide/install-guide/conf-checkin-checkout-title.md)」を参照してください。


**タグ表示の切り替え** - ![](images/Label_icon.svg)

タグは、要素の境界を示す視覚的なキューです。 エレメント境界は、エレメントの先頭と末尾を示します。 次に、これらの境界を視覚的なキューとして使用して、挿入ポイントを配置したり、境界内のテキストを選択したりできます。 文書内のエレメントの前後に別のエレメントを挿入する場合は、エレメントの開始境界または終了境界の前後に挿入ポイントを配置できます。

次のスクリーンショットは、タグビューがオンになっているドキュメントを示しています。

![](images/tags-view.png){width="650"}

タグビューがオンになっている文書では、次の操作を実行できます。

- **要素を選択**：要素の開始タグまたは終了タグをクリックして、そのコンテンツを選択します。

- **タグを展開または折りたたむ**: タグの「+」または「 – 」記号をクリックして、タグを展開または折りたたみます。

- **コンテキストメニューを使用**：コンテキストメニューには、選択した要素をカット、コピー、またはペーストするオプションが表示されます。 選択したエレメントの前または後にエレメントを挿入することもできます。 その他のオプションを使用すると、IDを生成したり、選択した要素のプロパティパネルを開いたりできます。

- **エレメントのドラッグ&amp;ドロップ**：エレメントのタグを選択し、ドキュメントに簡単にドラッグ&amp;ドロップできます。 ドロップ場所がエレメントが許可される有効な場所である場合、エレメントはドロップ場所に配置されます。


>[!NOTE]
>
> ユーザーがWeb エディターからタグビューを有効にした場合、セッション全体でも有効のままになります。 これは、後でタグビューにアクセスするために、もう一度タグビューを有効にする必要がないことを意味します。新規ユーザーのセッションのタグビューのデフォルト値は、ui\_config.json ファイルのtagsView プロパティによって決まります。 詳しくは、「Adobe Experience Manager Guides as a Cloud Serviceのインストールと設定」の「*タグビューのデフォルト値の設定*」セクションを参照してください。

**変更のトラックを有効/無効にする** ![](images/track-change-icon.svg)

「変更をトラック」モードを有効にすると、ドキュメントで行われたすべての更新を追跡できます。 変更履歴を有効にすると、すべての挿入と削除がドキュメントに取り込まれます。 削除されたすべてのコンテンツは取り消し線を使用して強調表示され、すべての挿入は緑色のテキストで強調表示されます。 さらに、トピックページの端にある改訂バーも表示されます。 繰り返しますが、削除されたコンテンツには赤いバーが表示され、追加されたコンテンツには緑のバーが表示されます。 同じ行に追加と削除がある場合は、緑と赤の両方のバーが表示されます。

次のスクリーンショットは、削除および挿入されたコンテンツと改訂バーを強調表示しています。

![](images/track-changes-content.png){width="650"}

ドキュメント内の変更を追跡する一般的な使用例は、ピアレビューを行う場合です。 変更履歴を有効にし、レビュー用にドキュメントを共有すると、レビュアーは変更履歴をオンにして変更を加えます。 ドキュメントを受け取ったら、変更を承認または却下する便利な方法とともに、提案された更新を表示するメカニズムが必要です。

AEM Guidesには、ドキュメントで行われた更新に関する情報を含む変更履歴の機能が用意されています。 「変更履歴」機能には、どのような更新が行われ、誰がその更新を行い、いつ更新されたかに関する情報が表示されます。 「変更履歴」機能を使用すると、ドキュメント内の提案された更新を簡単に承認または拒否できます。

この機能にアクセスするには、右側のパネルの「変更履歴」アイコンをクリックします。

![](images/changes-panel_cs.png){width="300"}

変更をクリックすると、文書内の変更されたコンテンツが選択されます。 変更を受け入れるには、「変更を受け入れる」アイコンを選択するか、「変更を却下」を選択して変更を却下します。

ワンクリックですべての変更を承認または拒否する場合は、「**すべて承認**」または「**すべて拒否**」を選択します。

>[!NOTE]
>
> プレビューモードでは、変更されたコンテンツのマークアップの有無にかかわらず、ドキュメントを表示できます。 詳しくは、[&#x200B; プレビュー](web-editor-views.md#preview-mode-id19AAGL00163) モードを参照してください。

**結合** - ![](images/merge-icon.svg)

マルチオーサー環境で作業する場合、トピックやマップ内で他のオーサーが行った変更を追跡するのは困難になります。 結合機能を使用すると、変更を表示するだけでなく、最新バージョンの文書に保持されている変更を制御できます。

**トピックファイルを結合**

トピックの変更を結合するには、次の手順を実行します。

1. Web エディターでトピックを開きます。

1. 「**結合**」をクリックします。

   結合ダイアログが表示されます。

   ![](images/merge-changes-in-topic.png){width="550"}

1. *\（Optional\）* リポジトリ内の別の場所から新しいファイルを参照して選択することもできます。

1. ファイルの現在のバージョンを比較するファイルのバージョンを選択します。

1. 「オプション」で、次を選択します。

   - **選択したバージョンからの変更履歴**：このオプションには、変更履歴の形式ですべてのコンテンツ更新が表示されます。 その後、文書内の変更を1回または1回で承認または却下できます。

   - **選択したバージョンに戻す**：このオプションは、ドキュメントの現在のバージョンを選択したバージョンに戻します。 このオプションでは、どのようなコンテンツが受け入れられるか、または拒否されるかを制御することはできません。

1. 「**完了**」をクリックします。

1. 「**選択したバージョンから変更を追跡**」オプションを選択した場合、選択したバージョンからのすべての変更が、右側のパネルの「変更履歴」機能に表示されます。

   変更履歴パネルからすべてのコメントを承認または却下するか、個々のコメントを承認または却下するかを選択できます。


**マップファイルを結合**

マップファイル内の変更を結合するには、次の手順を実行します。

1. Web エディターでマップを開きます。

1. 「**結合**」をクリックします。

   結合ダイアログが表示されます。

   ![](images/merge-changes-in-map.png){width="550"}

1. *\（Optional\）* リポジトリ内の別の場所から新しいファイルを参照して選択することもできます。

1. ファイルの現在のバージョンを比較するファイルのバージョンを選択します。

1. 「オプション」で、次を選択します。

   - **選択したバージョンからの変更履歴**：このオプションには、変更履歴の形式ですべてのコンテンツ更新が表示されます。 その後、文書内の変更を1回または1回で承認または却下できます。

   - **選択したバージョンに戻す**：このオプションは、ドキュメントの現在のバージョンを選択したバージョンに戻します。 このオプションでは、どのようなコンテンツが受け入れられるか、または拒否されるかを制御することはできません。

1. 「**完了**」をクリックします。

   1. 「**選択したバージョンから変更を追跡**」オプションを選択した場合、選択したバージョンからのすべての変更が、「（右側の）変更を追跡」パネルに表示されます。

      変更履歴パネルからのすべての変更を承認または却下するか、マップファイル内の個別の変更を承認または却下するかを選択できます。


**バージョン履歴** - ![](images/version-history-web-editor-ico.svg)


Web エディターの&#x200B;**バージョン履歴**&#x200B;機能を使用すると、DITA ファイルの使用可能なバージョンを確認し、それらを比較して、エディター自体から任意のバージョンに戻すことができます。

バージョン履歴では、現在のバージョンのコンテンツとメタデータを、同じファイルの以前のバージョンと比較できます（これは作業用コピーでもあります）。 比較したバージョンのラベルとコメントを表示することもできます。

バージョン履歴にアクセスし、トピックの特定のバージョンに戻すには、次の手順を実行します。

1. Web エディターでトピックを開きます。

1. 「**バージョン履歴**」をクリックします。

   **バージョン履歴** ダイアログボックスが表示されます。

   ![&#x200B; バージョン履歴ダイアログボックス](images/version-history-dialog-web-editor.png){width="550"}
   *トピックの様々なバージョンの変更をプレビューします。*

1. **比較または元に戻すトピックのバージョンを**&#x200B;と比較ドロップダウンリストで選択します。

   >[!NOTE]
   >
   > バージョンにラベルが適用されている場合は、バージョン番号と共に\（括弧で囲んだ\）も表示されます。



1. 「**ラベルとコメントを表示**」オプションを有効にして、現在のバージョンと比較されたバージョンに適用されたラベルとコメントを表示します。

1. **バージョン履歴** ダイアログボックスでは、次の情報も表示できます。

   **プレビュー** タブ：新しく追加されたコンテンツは緑色のフォントで、削除されたコンテンツは赤色のフォントです。

   **メタデータ** タブ：新しく追加されたメタデータは緑色のフォントで、削除されたメタデータは赤色のフォントです。
   バージョンの![&#x200B; メタデータの違い &#x200B;](images/metadata-version-diff.png){width="550"}
   *バージョン履歴の異なるバージョンのメタデータを比較します。*

   >[!NOTE]
   >
   > システム管理者は、エディター設定の「メタデータ」タブから表示するメタデータを変更できます。

   また、現在のバージョンと比較されたバージョンのユーザーと時間の詳細を表示することもできます。



1. ドロップダウンリストからバージョンを選択すると、**選択したバージョンに戻すオプション**&#x200B;が使用可能になります。 プレビューウィンドウには、現在のバージョンと選択したバージョンのトピックの違いが表示されます。


1. 「**選択したバージョンに戻す**」をクリックして、選択したバージョンのトピックで作業コピーを元に戻します。

   バージョンを元に戻すダイアログが表示されます。

   ![](images/version-history-revert-dialog-save-working-copy.png){width="550"}

**バージョンラベル管理** - ![](images/version-label-icon.svg)

ラベルは、特定のトピックがDDLC \（ドキュメント開発ライフサイクル\）内にあるステージを特定するのに役立ちます。 例えば、トピックに取り組んでいる場合、ラベルを「承認済み」に設定できます。 トピックが公開され、顧客が利用できるようになったら、そのトピックに「リリース済み」ラベルを割り当てることができます。

AEM Guidesでは、自由形式のテキストフォーマットでラベルを指定したり、定義済みのラベルのセットを使用したりできます。 カスタムラベルを使用すると、システム内の任意の作成者が、選択に従ってラベルを指定できます。 これにより柔軟性が得られますが、システム内に一貫性のないラベルが導入されます。 この問題を解決するために、管理者は定義済みのラベルのセットを設定できます。 定義済みラベルの設定について詳しくは、「Adobe Experience Manager Guides as a Cloud Serviceのインストールと設定」の「*XML Web Editorの設定とカスタマイズ」を参照してください。*

これらのラベルは、作成者がラベルを指定する必要がある場所に、ドロップダウンリストの形式で表示されます。 これにより、システムで事前定義された一貫性のあるラベルのみが使用されます。

Assets UIの[&#x200B; バージョン履歴](web-editor-use-label.md) パネル、[&#x200B; ベースライン &#x200B;](/help/product-guide/user-guide/generate-output-use-baseline-for-publishing.md) UIおよびWeb エディターなど、トピックにラベルを適用する方法は様々です。 Web エディターのバージョンラベル機能を使用すると、作成者はトピックにラベルを素早く簡単に割り当てることができます。

Web エディターからトピックにラベルを追加するには、次の手順を実行します。

1. Web エディターでトピックを開きます。

1. 「**バージョンラベル**」をクリックします。

   バージョンラベル管理ダイアログが表示されます。

   ![](images/version-label-management-dialog.png){width="650"}

   バージョンラベル管理ダイアログは2つの部分に分かれています。左側のパネルには、トピックで使用可能なバージョンのリストと、ラベルドロップダウンリスト \（またはラベルを入力するためのテキストボックス\）と、トピックのプレビューを含む右側のパネルがあります。

1. ラベルを適用するバージョンを選択します。

   バージョンリストから別のバージョンのトピックを選択すると、プレビューパネルに、現在のバージョンと選択したバージョンのトピックの間の変更点が表示されます

   >[!NOTE]
   >
   > ラベルが既にバージョンに適用されている場合は、ドロップダウンリストのバージョン番号の横と「バージョンを選択」リストの下に表示されます。 ラベルの横にある\（**x**\） アイコンをクリックすると、既存のラベルを削除できます。

1. 管理者がラベルのリストを定義している場合、適用するラベルを選択できるラベルのドロップダウンリストが表示されます。 ドロップダウンリストから複数のラベルを選択できます。

   または、テキストボックスが表示され、トピックに追加するラベルを入力できます。

   >[!NOTE]
   >
   > トピックの複数のバージョンに同じラベルを適用することはできません。 既存のラベルを関連付けようとした場合は、既存のバージョンからラベルを削除し、選択したバージョンのトピックに適用するオプションが表示されます。

1. 「**ラベルを追加**」をクリックします。

1. ラベルを適用の確認メッセージで、「**ラベルを移動**」オプションを選択して、ラベルを既存のバージョンから選択したバージョンに移動します。 このオプションを選択せず、トピックの別のバージョンに割り当てられているラベルがある場合は、選択したトピックのバージョンに移動されません。 このようなラベルは、ラベル適用プロセスでは無視されます。


**レビュータスクの作成** - ![](images/create-review-task-icon.svg)

Web エディターから直接、現在のトピックまたはマップファイルのレビュータスクを作成できます。 レビュータスクを作成するファイルを開き、「レビュータスクを作成」をクリックしてレビュー作成プロセスを開始します。

オプションメニューの様々なオプションについて説明します。

- **編集**：編集するファイルを開きます。 .ditamap/.bookmap ファイルの場合は、[詳細マップエディター](map-editor-advanced-map-editor.md#)で開いて編集します。

- **重複**：このオプションを使用して、選択したファイルの重複またはコピーを作成します。 また、アセットを複製プロンプトで複製ファイルの名前を変更するオプションもあります。 デフォルトでは、ファイルは接尾辞\（例えばfilename\_1.extension\）を付けて作成されます。 ファイルのタイトルはソースファイルと同じままであり、新しいファイルはバージョン 1.0で始まります。 ベースラインが重複ファイルにコピーされない間、すべての参照、タグ、メタデータがコピーされます。
- **チェックアウト**：選択したファイルを編集するためにロックします。 ロックされたファイルの場合、このオプションは&#x200B;**チェックイン**&#x200B;に変更されます。

  >[!NOTE]
  >
  > - ユーザーがファイルをロックまたはチェックアウトした場合、ロックアイコンの上にマウスポインターを置くと、ファイルをロックしたユーザー\（name\）が表示されます。
  > - 保存されていない変更があるファイルをチェックインすると、変更を保存するよう求められます。 変更を保存しない場合は、ファイル内でのみチェックされます。

- **プレビュー**: ファイル （.dita、.xml、オーディオ、ビデオ、または画像）を開かずに簡単にプレビューできます。 プレビューペインのサイズを変更できます。 コンテンツに`<xref>`または`<conref>`が含まれている場合は、コンテンツを選択して新しいタブで開くことができます。 ファイルのタイトルがウィンドウに表示されます。 タイトルがない場合は、ファイル名が表示されます。 **プレビュー** ペインを閉じるには、閉じるアイコンを選択するか、ペインの外側の任意の場所をクリックします。

  ![](images/quick-preview_cs.png){width="800"}

- **名前を変更**：このオプションを使用して、選択したファイルの名前を変更します。 「**アセット名を変更**」ダイアログに新しいファイルの名前を入力します。
   - 任意のタイプのファイル名を変更できます。
   - ファイルの拡張子は変更できません。
   - 2つのファイルに同じ名前を付けることはできません。 そのため、既に存在する名前にファイルの名前を変更することはできません。 エラーが表示されます。

- **移動先**：選択したファイルを別のフォルダーに移動するには、このオプションを使用します。
   - 宛先フォルダーの名前を入力するか、**パスを選択**&#x200B;して宛先フォルダーを選択します。
   - 任意のタイプのファイルをコンテンツフォルダー内の任意の宛先に移動できます。
   - 2つのファイルに同じ名前を付けることはできません。 そのため、同じ名前のファイルが既に存在するフォルダーにファイルを移動することはできません。

  同じ名前で別のタイトルのファイルが存在するフォルダーにファイルを移動しようとすると、「ファイル名を変更して移動」ダイアログが表示され、ファイルを移動する前にファイル名を変更する必要があります。 移動先フォルダー内の移動されたファイルには、新しいファイル名が付きます。

  ![](images/rename-move-asset.png){width="550"}

  >[!NOTE]
  >
  > ファイルを別の宛先フォルダーにドラッグ&amp;ドロップすることもできます。

  **除外シナリオ**

  AEM Guidesでは、次の場合、ファイルの名前を変更したり、ファイルを移動したりすることはできません。

   - ファイルがレビューまたは翻訳ワークフローの一部である場合、ファイルの移動や名前の変更はできません。

   - 他のユーザーがファイルをチェックアウトした場合、ファイルの名前を変更したり移動したりできない場合は、ファイルの「名前を変更」または「ファイルに移動」オプションは表示されません。

  >[!NOTE]
  >
  > 管理者がフォルダーに対する権限を与えた場合は、**名前の変更**&#x200B;または&#x200B;**移動先**&#x200B;のオプションのみが表示されます。

  <details>
    <summary> クラウドサービス </summary>

  ファイルの名前を変更したり移動したりしても、各ファイルには一意のUUIDがあるため、ファイルとの既存の参照が壊れることはありません。
  </details>



- **削除**：このオプションを使用して、選択したファイルを削除します。 ファイルを削除する前に、確認プロンプトが表示されます。

   - ファイルを削除する前に、確認プロンプトが表示されます。
   - 他のファイルから参照されていないファイルは削除され、成功メッセージが表示されます。
   - ファイルがチェックアウトされている場合は、ファイルを削除できません。エラーメッセージが表示されます。

     >[!NOTE]
     >
     > 管理者がチェックアウトしたファイルの削除を禁止している場合は、エラーメッセージのみが表示されます。 詳しくは、「Adobe Experience Manager Guides as a Cloud Serviceのインストールと設定」の「*チェックアウト済みファイルの削除を防止する*」セクションを参照してください。

   - ファイルをお気に入りコレクションに追加すると、**強制削除** ダイアログが表示され、強制的に削除できます。
   - ファイルが他のファイルから参照されている場合、確認メッセージを含む&#x200B;**強制削除** ダイアログが表示され、ファイルを強制的に削除できます。

     ![](images/options-menu-force-delete.png){width="550"}

     >[!NOTE]
     >
     > 管理者がファイルの削除権限を付与している場合、**強制削除**&#x200B;が有効になっています。 そうでない場合、**強制的な削除**&#x200B;は無効になり、参照ファイルを削除する権限がないことを示すメッセージが表示されます。 詳しくは、「Adobe Experience Manager Guides as a Cloud Serviceのインストールと設定」の「*参照ファイルの削除を禁止する*」セクションを参照してください。

   - 参照トピックを削除し、編集用の参照を含むファイルを開いた場合、参照ファイルの壊れたリンクが表示されます。

  >[!NOTE]
  >
  > キーボードのDelete キーを使用して、選択したファイルを同様に削除することもできます。

- **コピー**：次のオプションから選択できます。

   - **UUIDをコピー**：選択したファイルのUUIDをクリップボードにコピーします。

   - **パスをコピー**：選択したファイルの完全パスをクリップボードにコピーします。

- **すべてを折りたたむ**: リポジトリ内のすべてのファイルを折りたたみます。 リポジトリ内の最上位フォルダーのみが表示されます。
- **追加先**：次のオプションから選択できます。
   - **お気に入り**：選択したファイルをお気に入りに追加します。 既存のお気に入りコレクションまたは新しいお気に入りコレクションに追加できます。

   - **再利用可能なコンテンツ**：選択したファイルを左パネルの「再利用可能なコンテンツ」リストに追加します。

- **プロパティ**：これを使用して、選択したファイルのプロパティ ページを開きます。 このプロパティページには、Assets UIからファイルを選択し、ツールバーのプロパティアイコンをクリックしてアクセスすることもできます。

- **マップ ダッシュボードを開く**：選択したファイルがDITA マップの場合、このオプションを選択するとマップ ダッシュボードが開きます。

- **Oxygenで編集**:Oxygen コネクタプラグインで選択したファイルを編集するには、このオプションを選択します。 ファイルが編集用に開かれます。

  >[!NOTE]
  >
  >この機能を環境で有効にするには、カスタマーサクセス チームにお問い合わせください。 これは、すぐに使えるサポートの一部として有効になっていません。 詳細については、『インストールおよび設定ガイド』の「[Oxygen](/help/product-guide/cs-install-guide/conf-edit-in-oxygen.md)」セクションで編集するオプションを設定する」を参照してください。


- **Assets UI**&#x200B;で表示：これを使用すると、Assets UI で.dita/.xml ファイルのプレビューを表示できます。 .ditamap/.bookmap ファイルの場合、マップ内のすべてのトピックファイルは、単一の統一されたページごとのビューに表示されます。

- **PDFとしてダウンロード**：このオプションを使用して、PDF出力を生成し、ダウンロードします。

- **別名で公開**：このオプションを使用して、トピックまたはトピック内の要素をコンテンツフラグメントに公開します。

- **クイック生成**：選択したファイルの出力を生成します。 出力は、出力プリセットの一部であるファイルに対してのみ生成できます。 詳しくは、[Web エディターからの記事ベースの公開](web-editor-article-publishing.md#id218CK0U019I)を参照してください。


**リポジトリビューからトピックを作成**

リポジトリパネルの横にある+ アイコンまたはリポジトリビューのフォルダーのコンテキストメニューから、新しいトピック、マップ、フォルダーを作成できます。

***トピックの作成***

メニューから&#x200B;*新しいトピック*&#x200B;を作成することを選択すると、次のダイアログが表示されます。

![](images/create-topic-dialog.png){width="300"}

**新しいトピックを作成** ダイアログで、次の詳細を入力します。

- トピックのベースとなるテンプレート。 例えば、標準の設定では、「空白」、「概念」、「DITAVAL」、「参照」、「タスク」、「トピック」、「トラブルシューティング」の各テンプレートから選択できます。

  フォルダーにフォルダープロファイルが設定されている場合は、フォルダープロファイルで設定されているトピックテンプレートのみが表示されます。

- トピックファイルを保存するパス。 デフォルトでは、リポジトリで現在選択されているフォルダーのパスが「パス」フィールドに表示されます。
- トピックのタイトル。

- *\（Optional\）* トピックのファイル名。 ファイル名は、トピックタイトルに基づいて自動的に提案されます。

  管理者がUUID設定に基づいて自動ファイル名を有効にしている場合、次のスクリーンショットに示すように、「名前」フィールドが表示されません。

  ![](images/new-topic-without-filename.PNG){width="300"}


**作成**&#x200B;をクリックすると、トピックは指定されたパスに作成されます。 また、トピックは編集のためにWeb エディターで開かれます。

***DITA マップを作成***

新しいDITA マップを&#x200B;*作成*&#x200B;することを選択すると、次のダイアログが表示されます。

![](images/create-map-dialog.png){width="300"}

**新しいマップを作成** ダイアログで、次の詳細を入力します。

- マップのベースとなるテンプレート。 例えば、標準の設定では、ブックマップまたはDITA マップテンプレートから選択できます。

- マップファイルを保存するパス。 デフォルトでは、リポジトリで現在選択されているフォルダーのパスが「パス」フィールドに表示されます。
- マップの&#x200B;**タイトル**。

- *\（Optional\）* マップのファイル名。 ファイル名は、マップタイトルに基づいて自動的に提案されます。

  管理者がUUID設定に基づいて自動ファイル名を有効にしている場合、「名前」フィールドは表示されません。


**作成**&#x200B;をクリックすると、マップが作成され、「パス」フィールドで指定したフォルダー内に追加されます。 また、マップはマップビューで開きます。 マップエディターでマップファイルを開き、トピックを追加できます。 マップファイルへのトピックの追加について詳しくは、[&#x200B; マップの作成](map-editor-create-map.md#)を参照してください。

***フォルダーを作成***

*新しいフォルダー*&#x200B;を作成することを選択すると、**新しいフォルダーを作成** ダイアログが表示されます。

![](images/new-folder-dialog_cs.png){width="300"}

フォルダーの&#x200B;**タイトル**&#x200B;を入力すると、フォルダー名に自動変換されます。 パスは、マップファイルを保存する場所です。 デフォルトでは、リポジトリで現在選択されているフォルダーのパスが「パス」フィールドに表示されます。 「**作成**」をクリックすると、フォルダーが作成され、「フォルダーを作成」オプションが実行されたフォルダー内に追加されます。

**マップ ビュー** - ![](images/map-view-icon.svg)

マップビューアイコンをクリックすると、マップファイル内のトピックのリストが表示されます。 マップファイルを開いていない場合は、マップビューが空白で表示されます。 任意のマップファイルをダブルクリックすると、このビューでマップファイルが開きます。 マップ内の任意のファイルをダブルクリックして、Web エディターで開くことができます。

デフォルトでは、タイトルでファイルを表示できます。 ファイルにカーソルを合わせると、ファイルタイトルとファイルパスがツールチップとして表示されます。

>[!NOTE]
>
>管理者は、マップビューで現在開いている親マップのファイル名を表示することもできます。 **ユーザー設定** ![](images/user_preference_editor_icon.svg)の「**でファイルを表示」セクションの「** ファイル名&#x200B;**」オプションを選択します。**


マップビューでマップを開くと、現在のマップのタイトルがメインツールバーの中央に表示されます。 タイトルが長すぎる場合は、省略記号が表示され、タイトルにカーソルを合わせると、ツールヒントにタイトル全体が表示されます。

トピックまたはマップ参照のキー属性を定義すると、タイトル、対応するアイコン、および左側のパネルのキーを表示できます。 キーは`keys=<key-name>`として表示されます。

マップビュー![&#128279;](images/view-key-title-map-view.png){width="300"}の キー

マップファイルの編集権限がある場合は、ファイルも編集できます。 DITA マップを使用してトピックを開く方法と編集する方法について詳しくは、[DITA マップを使用してトピックを編集](map-editor-advanced-map-editor.md#id17ACJ0F0FHS)を参照してください。


マップファイルのオプションメニューを使用して、次のアクションを実行できます。

![](images/options-menu-map-view_cs.png){width="550"}

- **編集**：詳細マップエディターで編集するマップファイルを開きます。

- **すべてを選択**: マップ内のすべてのファイルを選択します。

- **選択範囲をクリア**: マップ内の選択したファイルの選択を解除します。

- **チェックアウトとロック**：チェックアウトして、マップ内の選択したファイルのロックを取得します。

- **チェックアウトをキャンセルしてロックを解除**：マップファイルのロックを解除し、編集可能にします。 変更を以前のバージョンに戻すことはありません。

- **新しいバージョンとして保存してロックを解除**：新しいバージョンを作成し、マップ内の選択したファイルのロックを解除します。

- **プレビュー**: マップファイルのプレビューを開きます。 このビューでは、マップ内のすべてのトピックファイルが、単一の統一されたページ単位のビューで表示されます。

- **コピー**：次のオプションから選択できます。
   - **UUIDをコピー**: マップファイルのUUIDをクリップボードにコピーします。
   - **パスをコピー**: マップファイルの完全パスをクリップボードにコピーします。

- **リポジトリ内の場所**: リポジトリ \（またはDAM\）内のマップファイルの場所を表示します。

- **追加先**：次のオプションから選択できます。
   - **お気に入り**: マップファイルをお気に入りに追加します。 既存のお気に入りコレクションまたは新しいお気に入りコレクションに追加できます。

   - **再利用可能なコンテンツ**：左パネルの「再利用可能なコンテンツ」リストにマップファイルを追加します。

- **プロパティ**：これを使用して、マップファイルのプロパティページを開きます。 このプロパティページには、Assets UIからファイルを選択し、ツールバーのプロパティアイコンをクリックしてアクセスすることもできます。

- **マップ ダッシュボードを開く**: マップ ダッシュボードを開きます。

- **Assets UI**&#x200B;で表示：これを使用すると、Assets UIでマップファイルのプレビューを表示できます。 このビューでは、マップ内のすべてのトピックファイルが、単一の統一されたページ単位のビューで表示されます。
- **マップをダウンロード**：このオプションを選択すると、**マップをダウンロード** ダイアログボックスが開きます。

**マップをダウンロード** ダイアログボックスで、次のオプションを選択できます。

- **ベースラインを使用**: DITA マップ用に作成されたベースラインのリストを取得するには、このオプションを選択します。 特定のベースラインに基づいてマップファイルとその内容をダウンロードする場合は、ドロップダウンリストから「ベースライン」を選択します。 ベースラインの操作について詳しくは、[&#x200B; ベースラインの操作](./generate-output-use-baseline-for-publishing.md)を参照してください。
- **ファイル階層を統合**：参照されているすべてのトピックとメディアファイルを1つのフォルダーに保存するには、このオプションを選択します。

  オプションを選択せずにマップファイルをダウンロードすることもできます。 この場合、参照されたトピックとメディアファイルの最後の永続バージョンがダウンロードされます。

  「**ダウンロード**」ボタンをクリックすると、マップ書き出しパッケージリクエストがキューに入れられます。 パッケージが正常に作成されると、**成功** ダイアログボックスが表示されます。  **成功** ダイアログボックスから「**ダウンロード**」ボタンをクリックできます。

  マップをダウンロードする準備ができたら、マップのダウンロード準備通知を受け取ります。 ダウンロードに失敗した場合、マップのダウンロードに失敗したという通知が表示されます。

  ダウンロードリンクには、AEMの通知インボックスからアクセスできます。 インボックスで生成されたマップ通知を選択して、マップを.zip形式でダウンロードします。

  >[!NOTE]
  >
  >  デフォルトでは、ダウンロードされたマップはAEM通知インボックスに5日間残ります。

- **出力を生成**：選択したマップファイルの出力を生成します。 出力は、出力プリセットの一部であるファイルに対してのみ生成できます。 詳しくは、[Web エディターからの記事ベースの公開](web-editor-article-publishing.md#id218CK0U019I)を参照してください。
- **閉じる**: マップファイルを閉じます。



次のスクリーンショットは、DITA マップビューのファイルのオプションメニューを示しています。

![](images/options-menu-file_cs.PNG){width="550"}

オプションメニューを使用して、次のアクションを実行できます。

- **編集**：編集するファイルを開きます。 .ditamap/.bookmap ファイルの場合は、[詳細マップエディター](map-editor-advanced-map-editor.md#)で開いて編集します。

- **チェックアウト**：選択したファイルをチェックアウトします。 チェックアウト済みファイルの場合、このオプションは&#x200B;**チェックイン**&#x200B;に変更されます。



  >[!NOTE]
  >
  > - ユーザーがファイルをロックまたはチェックアウトした場合、ロックアイコンの上にマウスポインターを置くと、ファイルをロックしたユーザー\（name\）が表示されます。
  > - ファイルをチェックインすると、変更内容の保存を求めるメッセージが表示されます。 変更を保存しない場合は、ファイル内でのみチェックされます。

- **プレビュー**: ファイル （.dita、.xml、オーディオ、ビデオ、または画像）を開かずに簡単にプレビューできます。 プレビューペインのサイズを変更できます。 コンテンツに`<xref>`または`<conref>`が含まれている場合は、コンテンツを選択して新しいタブで開くことができます。  ファイルのタイトルがウィンドウに表示されます。 タイトルがない場合は、ファイル名が表示されます。 **プレビュー** ペインを閉じるには、閉じるアイコンを選択するか、ペインの外側の任意の場所をクリックします。
- **コピー**：次のオプションから選択できます。
   - **UUIDをコピー**：選択したファイルのUUIDをクリップボードにコピーします。
   - **パスをコピー**：選択したファイルの完全パスをクリップボードにコピーします。


- **リポジトリ内の場所**: リポジトリ \（またはDAM\）内の選択したファイルの場所を表示します。
- **すべてを展開**: マップファイル内のすべてのトピックを展開します。

- **すべてを折りたたむ**：現在のマップファイルの一部であるすべてのトピックを折りたたみます。

- **追加先**：次のオプションから選択できます。
   - **お気に入り**：選択したファイルをお気に入りに追加します。 既存のお気に入りコレクションまたは新しいお気に入りコレクションに追加できます。

   - **再利用可能なコンテンツ**：選択したファイルを左パネルの「再利用可能なコンテンツ」リストに追加します。

- **プロパティ**：これを使用して、選択したファイルのプロパティ ページを開きます。 このプロパティページには、Assets UIからファイルを選択し、ツールバーのプロパティアイコンをクリックしてアクセスすることもできます。

- **Assets UI**&#x200B;で表示：これを使用すると、Assets UI で.dita/.xml ファイルのプレビューを表示できます。 .ditamap/.bookmap ファイルの場合、マップ内のすべてのトピックファイルは、単一の統一されたページごとのビューに表示されます。

- **クイック生成**：選択したファイルの出力を生成します。 出力は、出力プリセットの一部であるファイルに対してのみ生成できます。 詳しくは、[Web エディターからの記事ベースの公開](web-editor-article-publishing.md#id218CK0U019I)を参照してください。

>[!NOTE]
>
> また、マップビューの下部にある&#x200B;**詳細オプション** メニューから、DITA マップ内の選択したトピックのプロパティを開いて編集することもできます。

**アウトラインビュー** - ![](images/outline-icon.svg)

アウトラインビューには、次の機能があります。

- ドキュメントで使用されているすべての要素のツリービュー。

- エレメントにID、属性およびテキストがある場合、エレメントと共にそれらを確認できます。

- 作成者ビューとSource ビューの両方でアウトライン ビューにアクセスします。

- フィルタードロップダウンリストを使用して、すべての要素を表示するか、壊れた参照のみを表示します。

- アウトラインビューでエレメントをクリックすると、オーサービューまたはSource ビューでエレメントの内容が選択されます。アウトラインビューは、オーサービューとSource ビューと同期したままになります。 任意のビューで変更を加えた場合は、アウトライン表示で変更を確認できます。 例えば、作成者ビューで段落を追加したりエレメントを更新したりすると、アウトライン表示に表示されます。

  ![](images/select-element-content-outline-view_cs.png){width="650"}

- エレメントをドラッグ&amp;ドロップします。 エレメントに別のエレメントをドロップすることで、エレメントを簡単に置き換えることができます。 エレメントを別のエレメントの上にドラッグ&amp;ドロップすると、そのエレメントの周囲に正方形のボックスが表示され、そのエレメントが置き換えられることを示します。 要素がドロップされる要素が置き換えられます。

  ![](images/replace-element-outline-view_cs.png){width="300"}

  エレメントをドラッグ&amp;ドロップすると、破線の長方形は、エレメントを現在の位置に配置できることを示します。 ドラッグ&amp;ドロップが無効な場合、操作が許可されていないことを示すエラーメッセージが表示されます。

  ![](images/drop-element-outline-view_cs.png){width="300"}

- *アウトラインビュー*&#x200B;の&#x200B;**オプション** メニューでは、カット、コピー、削除、ID生成、現在の要素の前後に要素を挿入、要素の名前変更または置換、要素のラップ解除、選択した要素からのスニペットの作成などの汎用操作を実行できます。

>[!NOTE]
>
>IDの生成、現在の要素の前後に要素を挿入、要素のラップ解除について詳しくは、[Web エディターのその他の機能](web-editor-other-features.md#)を参照してください。

**アウトライン表示パネルの表示オプション**

表示オプション ドロップダウンを使用して、要素に次のオプションがある場合に、そのオプションを表示できます。

- **Show ID**：要素のIDを表示します。
- **属性を表示**：属性とその値を表示します。
- **テキストを表示**: テキストを表示します。 テキストが20文字を超える場合、省略記号が表示されます。

ブロック要素に独自のテキストがある場合は、そのブロック要素と共に表示されます。 独自のテキストがない場合は、最初の子要素のテキストがブロック要素と共に表示されます。

![](images/outline-view-block-element.png){width="550"}

管理者が属性のプロファイルを作成している場合は、これらの属性と設定値を取得します。 管理者が設定した表示属性は、エディター設定の「**表示属性**」タブで割り当てることもできます。 エレメントに対して定義された属性は、レイアウトおよびアウトライン表示に表示されます。


詳細については、[左パネル &#x200B;](web-editor-features.md#id2051EA0M0HS) セクションの&#x200B;*エディター設定*&#x200B;機能の説明にある&#x200B;*表示属性*&#x200B;を参照してください。

**検索機能**
検索機能を使用すると、名前、ID、テキストまたは属性値でエレメントを検索できます。

検索では大文字と小文字が区別されず、文字列と完全に一致します。 検索結果は、文書内の要素の位置に基づいてソートされます。

アウトライン表示パネルに表示されているエレメント内の文字列を検索できます。 例えば、「Adobe」という文字列がエレメントのテキストに存在し、アウトライン表示パネルに表示されている場合（表示オプション ドロップダウンから「**テキストを表示**」を選択したように）、含まれるエレメントがフィルタリングされます。 ただし、アウトライン表示パネルにテキストが表示されない場合（表示オプション ドロップダウンから「**テキストを表示**」を選択していないため）、含まれる要素はフィルタリングされません。 同様に、IDまたは属性で文字列を選択した場合は、文字列が見つかります。



**再利用可能なコンテンツ** - ![](images/content-reuse-icon.png)

DITAの主な機能の1つは、コンテンツを再利用できることです。 再利用可能なコンテンツパネルには、再利用可能なコンテンツを挿入する場所からDITA ファイルを格納できます。 追加すると、DITA ファイルはセッション間で再利用可能なコンテンツパネルに残ります。 つまり、後でアクセスするためにDITA ファイルを再度追加する必要はありません。

再利用可能なコンテンツをパネルから現在のトピックにドラッグ&amp;ドロップするだけで、簡単かつ迅速に挿入できます。 ドキュメントに挿入する前に、コンテンツのプレビューを表示することもできます。

デフォルトでは、タイトルでファイルを表示できます。 ファイルにカーソルを合わせると、ファイルタイトルとファイルパスがツールチップとして表示されます。

>[!NOTE]
>
> 管理者は、Web エディターでファイル名でファイルのリストを表示することもできます。 **ユーザー設定** ![](images/user_preference_editor_icon.svg)の「**でファイルを表示」セクションの「** ファイル名&#x200B;**」オプションを選択します。**

DITA ファイルを再利用可能なコンテンツパネルに追加するには、次のいずれかの方法を使用します。

- 「再利用可能なコンテンツ」の横にある「+」アイコンをクリックして、ファイルを参照ダイアログを開きます。 追加するファイルを選択し、**追加**&#x200B;をクリックしてプロセスを完了します。

  ![](images/reuse-content-add-dita-file_cs.png){width="650"}

- リポジトリビューで、目的のファイルのオプションアイコンをクリックし、コンテキストメニューから「**再利用可能なコンテンツに追加**」を選択します。

- エディターのファイルのタブを右クリックしてコンテキストメニューを開き、**再利用可能なコンテンツに追加**&#x200B;を選択します。


ファイルを追加すると、再利用可能なコンテンツパネルに、ファイルからすべての再利用可能なコンテンツ要素が表示されます。 再利用可能なコンテンツが、IDと要素名で表示されます。

再利用可能なコンテンツ リストにファイルを追加すると、ファイルのUUIDではなく、ファイルのタイトルが表示されます。 ファイルのUUIDを確認するには、ファイルのタイトルにマウスポインターを置くと、ファイルのUUIDがツールチップに表示されます。

![](images/uuid-reusable-content-file-title_cs.png){width="300"}

>[!NOTE]
>
> 再利用可能なコンテンツリストに複数のファイルを追加できます。 次に、再利用可能なコンテンツ パネルから目的のコンテンツをドキュメントに挿入できます。

**更新**：再利用可能なすべてのコンテンツを再確認し、再利用可能なコンテンツの新しいリストを表示します。

再利用可能なコンテンツパネルからコンテンツを挿入するには、次のいずれかの方法を使用します。

- 挿入する要素の上にマウスポインターを置き、「オプション」アイコンをクリックして、**再利用可能なコンテンツを挿入**&#x200B;を選択します。

  ![](images/insert-reusable-content_cs.png){width="400"}

  >[!NOTE]
  >
  > ファイルを選択し、**オプション** メニューから&#x200B;**プレビュー**&#x200B;を選択して、ファイルを開かずにプレビューします。 トピックに存在する参照をプレビューすることもできます。 参照IDがウィンドウに表示されます。
  >
  > **Preview** オプションは、エレメントの&#x200B;**Options** メニューでも使用できます。これにより、エレメントを挿入する前に、そのエレメントをすばやくプレビューできます。

- 再利用可能なコンテンツ項目を、ドキュメント内の目的の場所のパネルからドラッグ&amp;ドロップします。



**用語集** - ![](images/glossary.svg)

AEM Guidesを使用すると、用語集タイプのドキュメントを簡単に作成して使用できます。 用語集トピックファイルを作成し、共通の用語集マップに含めることができます。 このマップがルートマップとして追加されると、用語集エントリが用語集パネルに表示されます。

次の手順を実行して、テキスト用語を検索し、用語集の略語に置き換えます。

1. テキストまたは用語を検索して変換するDITA トピックまたはマップを開きます。

1. 用語集パネルで&#x200B;**ホットスポット** ツール \（![](images/hotspot-icon.svg)\）を選択して、特定のテキスト用語を検索し、リンクされた用語集の略語に変換します。 また、逆に、用語集の略語を検索してテキスト用語に変換する場合にも使用できます。

![](images/glossary-hotspot-tool.png){width="300"}

ホットスポットツールの次の設定を行うことができます。

![](images/Glossary-search-keys.png){width="300"}

- **用語集キー**：選択したトピックの検索に使用するDITA マップから用語集キーを選択します。 選択したキーが下に表示されます。 選択したキーを削除するには、**削除** アイコンをクリックします。

- **トピック**: Web エディターで開いた&#x200B;**現在のトピック**、現在のマップで開かれたすべての&#x200B;**開かれたトピック**、またはマップエディターで編集されている&#x200B;**現在のマップ**&#x200B;のいずれかを選択して、用語を検索します。
- **ステータス別にトピックをフィルタリング**：選択したドキュメントのステータスを持つトピックに検索を制限するように選択できます。 トピックは、ドラフト、編集、レビュー中、承認済み、レビュー済み、完了ステータス、または組織が設定した状態のいずれかになります。
- **アクション**：すべてのトピック **について、用語集キー**&#x200B;手動で各トピック **を検索するか、**&#x200B;自動的に検索するかを選択できます。 各トピック **に対して**&#x200B;手動を選択すると、各トピックの各用語を変換する前に、確認を求めるメッセージが表示されます。 すべてのトピックに対して&#x200B;**自動的に**&#x200B;を選択すると、すべてのトピックのすべての用語が自動的に変換されます。
- **変換**：検索した&#x200B;**テキストを用語集**&#x200B;に、または&#x200B;**用語集をテキストに変換できます。**
- **オプション**：次のオプションから選択できます。
   - **大文字と小文字を区別する一致**：同じ大文字と小文字が区別される一致を検索します。 例えば、「USB」は「usb」と一致しません。
   - **最初のインスタンスのみを変換**：検索された用語の複数のインスタンスがトピックに存在する場合、最初のインスタンスのみが変換されます。
   - **変換前にファイルをチェックアウト**：検索したファイルは、条件が変換される前にチェックアウトされます。
   - **変換後に新しいバージョンを作成**：用語の変換が完了した後に、トピックの新しいバージョンが作成されます。
- 「**各トピックに対して手動で**」オプションを選択すると、**次へ** ボタンが表示されます。 「**次へ**」をクリックして、選択した設定に基づいて各トピックの条件を変換します。 各トピックの用語の変換を求め、次のファイルに移動します。 用語を変換するか、それをスキップして次の用語に移動するかを選択できます。

  ![](images/manual-convert-skip.png){width="300"}

- 「すべてのトピック **」オプションで「**&#x200B;自動的に」を選択すると、**変換** ボタンが表示されます。 「**変換**」を選択して、ドキュメント内のすべての用語をリンクされた用語集の略語に変換します。

変換された用語を含む&#x200B;**更新済み** トピックと、エラー&#x200B;**を含む** トピックのリストが表示されます。 エラーのあるトピックの近くにある\（![](images/info-icon.svg)\） アイコンにカーソルを合わせると、エラーの詳細が表示されます。

![](images/glossary-converted-terms-error.png){width="300"}

**条件** - ![](images/conditions-icon.svg)

条件パネルには、管理者がグローバルレベルまたはフォルダーレベルのプロファイルで定義した条件付き属性が表示されます。 目的の条件をコンテンツにドラッグアンドドロップするだけで、コンテンツに条件を追加できます。 条件コンテンツは、簡単に識別できるように条件に定義された色を使用してハイライト表示されます。

エレメントに複数の条件をドラッグ&amp;ドロップして、エレメントに複数の条件を適用することもできます。 エレメントに複数の条件を適用すると、適用された条件がコンマで区切られてプロパティパネルに表示されます。

![](images/multiple-conditions-applied_cs.png){width="800"}

ただし、コードビューでは、条件はスペース区切り文字を使用して区切られます。 コードビューで条件を追加または編集する場合は、複数の条件がスペースで区切られていることを確認します。

>[!IMPORTANT]
>
> 次のスクリーンショットは、管理者権限を持つユーザーのものです。 管理者権限を持つユーザーは、条件を追加、編集、削除できます。 それ以外の場合は、通常の作成者として、条件を適用するオプションのみが表示されます。

![](images/conditional-content-through-panel_cs.png){width="800"}

![](images/conditional-panel-create-cond.png){width="400"}

属性リストから、定義する条件付き属性を選択し、条件の値を入力してから、条件パネルに表示されるラベルを指定します。 条件のグループを定義します。 グループに複数の条件を追加できます。 条件のカラーを定義することもできます。 この色は、条件が適用されるコンテンツの背景色として設定されます。

条件をグループ化し、ネストされたフォルダーに構造化できます。 グループは、複数レベルで条件を作成し、コンテンツでの使用に適して整理するのに役立ちます。

例えば、*AEM Guides*&#x200B;や&#x200B;*Acrobat*&#x200B;などの商品の条件グループを作成できます。 両方のグループの条件付き属性を選択できます。 各グループでは、*User*、*Admin*、*Reviewer*、*Author*&#x200B;など、特定の値を指定できます。

>[!NOTE]
>
> を入力して新しいグループを作成するか、特定の属性に既存のグループを選択します。

`/`を使用して、`AEM Guides/Cloud Service`のようなサブグループを定義するサブグループを定義できます。

**件名スキーム** - ![](images/subject_scheme_panel-icon.svg)

件名スキームマップは、分類の対象と制御値を定義するために使用されるDITA マップの特殊な形式です。 必要に応じて、件名スキームマップを作成し、ルートマップファイル内で参照できます。 AEM Guidesを使用すると、件名スキームの件名定義のネスト レベル階層を定義できます。

件名スキームマップでは、件名スキーマを簡単に作成して使用できます。 このマップをルートマップとして追加すると、件名スキームが件名スキームパネルに表示されます。 被写体スキーム パネルには、使用可能な被写体スキームがネストまたは階層的に表示されます。

AEM Guidesでは、ネストされたレベルのサブジェクトスキームのマップもサポートしており、ルートのサブジェクトスキームのマップの下で複数のサブジェクトスキームを定義できます。

次の例は、AEM Guidesで件名スキームを使用する方法を示しています。

1. 選択したツールで件名スキーム ファイルを作成します。 次のXML コードは、`platform`属性の値をバインドする件名スキームを作成します。

   ```XML
   <?xml version="1.0" encoding="UTF-8"?>
   ```

**件名の定義と列挙の階層定義の処理**

AEM Guidesでは、同じマップに存在する列挙と件名を処理するだけでなく、2つの個別のマップで列挙と件名を定義する機能も提供しています。 マップ内で1つ以上の件名の定義を定義し、別のマップ内で列挙定義を定義してから、マップ参照を追加できます。 例えば、次のXML コードは、2つの個別のマップで件名の定義と列挙定義を作成します。

```XML
        </subjectdef>
        <subjectdef keys="suse" navtitle="SuSE Linux">
        </subjectdef>
        </subjectdef>
        <subjectdef keys="windows" navtitle="Windows">
        </subjectdef>
        <subjectdef keys="zos" navtitle="z/OS">
        </subjectdef>
        </subjectdef>
        <subjectdef keys="deliveryTargetValues">
        <subjectdef keys="print">
        </subjectdef>
        <subjectdef keys="online">
        </subjectdef>
    </subjectdef>
```

```XML
    ?xml version="1.0" encoding="UTF-8"?> 
        <!DOCTYPE subjectScheme PUBLIC "-//OASIS//DTD DITA Subject Scheme Map//EN" "../dtd/libs/fmdita/dita_resources/DITA-1.3/dtd/subjectScheme/dtd/subjectScheme.dtd"> 
        <subjectScheme id="subject-scheme.ditamap_17c433d9-0558-44d4-826e-3a3373a4c5ae">

        </mapref>

        </attributedef>
        <subjectdef keyref="mobile">
        </subjectdef>

        </subjectdef>
        </enumerationdef>
        </subjectScheme>
```

ここでは、列挙定義が`subject_scheme_map_2.ditamap`に存在する間、`subject_scheme_map_1.ditamap`で件名の定義が定義されます。 `subject_scheme_map_1.ditamap`への参照も`subject_scheme_map_2.ditamap`に追加されています。

>[!NOTE]
>
> `subject_scheme_map_1.ditamap`と`subject_scheme_map_2.ditamap`が互いに参照されているため、件名スキームが解決されています。

サブジェクト列挙の参照は、次の優先順位で解決されます。

1. 同一マップ
1. 参照マップ


列挙が同じマップと参照マップに見つからない場合、参照は解決されません。

```XML
    </subjectdef>

    </subjectdef>
  </enumerationdef>
```

**スニペット** - ![](images/insert-snippet-icon.svg)

スニペットは、ドキュメントプロジェクトの様々なトピックで再利用できる小さなコンテンツフラグメントです。 スニペットパネルには、作成したコンテンツスニペットのコレクションが表示されます。 スニペットを挿入するには、スニペットをパネルからトピック内の目的の場所にドラッグ&amp;ドロップします。 スニペットパネルでは、スニペットを追加、編集、削除、プレビュー、挿入できます。

- コンテンツ編集領域で、スニペットとして使用するコンテンツの任意の場所を右クリックし、コンテキストメニューから「**スニペットを作成**」を選択します。 新規スニペット ダイアログが表示され、選択した要素のXML コードが&#x200B;**コンテンツ** フィールドに入力されます。 スニペットの&#x200B;**タイトル**&#x200B;と&#x200B;**説明**&#x200B;を入力し、**作成**&#x200B;をクリックしてスニペットを保存します。

**レビュー** - ![](images/active-review-tasklist-icon.svg)

AEM Guidesには、プロジェクト内のすべてのレビュータスクを表示する機能が用意されています。 **レビュー** パネルから参加しているすべてのレビュープロジェクトとアクティブなレビュータスクをレビュープロジェクト内で表示できます。  その後、レビュータスクを開いて、様々なレビュー担当者のコメントを表示できます。

レビューパネルにレビュータスクが表示されます。 デフォルトでは、タイトルでファイルを表示できます。 ファイルにカーソルを合わせると、ファイルタイトルとファイルパスがツールチップとして表示されます。

   - 「![](images/filter-search-icon.svg)」を選択して、**フィルター** ダイアログを開きます。 すべてのプロジェクトまたは特定のプロジェクトのみを選択できます。 選択したプロジェクトは、**レビュー** パネルに一覧表示されます。
     ![](images/active-review-select-project.png){width="300"}

     自分が開始した&#x200B;**タスク** オプションは、デフォルトで有効になっています。 開始したタスクのみを表示できます。 このオプションの切り替え状態は、ページを更新した後も維持されます。

1. デフォルトでは、レビュープロジェクトでは、コメントが関連付けられているトピックのフラットリストが表示されます。 左側のパネルから必要なフィルターを適用して、表示されるレビューコメントに基づいてトピックをフィルターします。

   - **コメント付きのトピックを表示**：レビューコメントを含むトピックのみを一覧表示します。
1. トピックのタイトルまたはファイルパスに、検索する検索語句またはテキストを入力することもできます。 タイトルまたはファイルパスに用語を含むトピックが一覧表示されます。

**検索と置換** - ![](images/FindAndReplace_icon.svg)

検索と置換アイコンは、左側のパネルの下部にあります。 検索と置換パネルを使用すると、マップまたはリポジトリ内のフォルダー内のファイル間でテキストを検索して置換できます。 マップのすべてのトピックと、マップ内のサブマップに存在するトピックを検索して置換できます。

![](images/map-find-replace.png){width="800"}

デフォルトでは、タイトルでファイルを表示できます。 ファイルにカーソルを合わせると、ファイルタイトルとファイルパスがツールチップとして表示されます。

>[!NOTE]
>
> 管理者は、Web エディターでファイル名のリストを表示することもできます。 **ユーザー設定** ![](images/user_preference_editor_icon.svg)の「**でファイルを表示」セクションの「** ファイル名&#x200B;**」オプションを選択します。**

グローバル検索と置換を実行するには、次の手順を実行します。

1. グローバルな&#x200B;**検索と置換** パネルを開きます。
1. 「**調べる**」ドロップダウンをクリックし、次のいずれかのオプションを選択して検索を実行します。

   - **現在のマップ**：現在開いているマップを検索するには

     >[!NOTE]
     >
     > このオプションは、編集用にマップを既に開いている場合に表示されます。

   - **パス**：選択したパスを検索するには
   - **マップを選択**：選択したマップで検索するには

1. 「**オプション**」ドロップダウンをクリックして、次のオプションから選択できます。

   - **置換前にファイルをチェックアウト**：検索語を置き換える前に自動的にファイルをチェックアウトする場合は、このオプションを選択します。 この設定は、管理者が編集前にファイルをチェックアウトする設定を有効にしている場合に適しています。 バックエンド設定が有効になっている場合は、このオプションを選択する必要があります。 ファイルのチェックアウトダイアログでは、変更を加える前に、すべてのファイルをチェックアウトするよう求めるメッセージが表示されません。 このオプションを選択しない場合は、ファイルを編集するために開く前にプロンプトが表示されます。
   - **全単語のみ**：検索文字列全体を検索する場合は、このオプションを選択します。 例えば、検索文字列にオーバーを入力した場合、検索結果には「オーバー」や「概要」などの単語を含むすべてのファイルが返されます。 入力された正確な語句を返すように検索を制限する場合は、このオプションを選択します。
   - **置換後に新しいバージョンを作成**：テキストを置換するトピックの新しいバージョンを作成する場合は、このオプションを選択します。 また、更新されたファイルごとに追加されるバージョンコメントを指定することもできます。

     このオプションを選択しない場合、変更はトピックの現在のバージョンに保存され、新しいバージョンは作成されません。

   - **間接参照を含める**: DITA マップ内でも間接参照の文字列を検索する場合は、このオプションを選択します。 デフォルトでは、これは無効になっているので、検索は直接参照に対してのみ実行されます。

1. 検索する検索語句またはテキストを入力します。
1. 検索語を置き換えるテキストを入力します。
1. Enter キーを押すか、**検索** アイコン \（![](images/search-icon.svg)\）を選択して検索を実行します。

   - **現在のマップ**：現在開いているマップを検索するには

     >[!NOTE]

   - **パス**：選択したパスを検索するには
   - **マップを選択**：選択したマップで検索するには

1. 「**オプション**」ドロップダウンをクリックして、次のオプションから選択できます。

   - **置換前にファイルをチェックアウト**：検索語を置き換える前に自動的にファイルをチェックアウトする場合は、このオプションを選択します。 この設定は、管理者が編集前にファイルをチェックアウトする設定を有効にしている場合に適しています。 バックエンド設定が有効になっている場合は、このオプションを選択する必要があります。 ファイルのチェックアウトダイアログでは、変更を加える前に、すべてのファイルをチェックアウトするよう求めるメッセージが表示されません。 このオプションを選択しない場合は、ファイルを編集するために開く前にプロンプトが表示されます。

   - **全単語のみ**：検索文字列全体を検索する場合は、このオプションを選択します。 例えば、検索文字列にオーバーを入力した場合、検索結果には「オーバー」や「概要」などの単語を含むすべてのファイルが返されます。 入力された正確な語句を返すように検索を制限する場合は、このオプションを選択します。

   - **置換後に新しいバージョンを作成**：テキストを置換するトピックの新しいバージョンを作成する場合は、このオプションを選択します。 また、更新されたファイルごとに追加されるバージョンコメントを指定することもできます。

     このオプションを選択しない場合、変更はトピックの現在のバージョンに保存され、新しいバージョンは作成されません。

   - **間接参照を含める**: DITA マップ内でも間接参照の文字列を検索する場合は、このオプションを選択します。 デフォルトでは、これは無効になっているので、検索は直接参照に対してのみ実行されます。

1. 「**単一の出現を置換** \（![](images/replace-icon.svg)\）」をクリックして、トピック内で現在強調表示されている検索語句を置換するか、「次の一致![](images/next-match-in-search.png)」または「前の一致![](images/previous-match-in-search.png)」をクリックして、テキストの次または前の出現に移動します。
1. 「**ファイル内のすべてを置換** \（![](images/replace-all-in-file-icon.svg)\）をクリックすると、1回のクリックで検索された用語のすべての出現箇所を、置換された用語に置き換えることができます。 選択したファイルのすべてのオカレンスを置き換えた後に通知が表示されます。

   >[!NOTE]
   >
   > 検索結果リストのファイルにカーソルを合わせると、その右側にある「ファイル内のすべてを置換」アイコンが表示されます。 ファイルを無視アイコンを使用して、検索結果からファイルを削除することもできます。 無視したファイルはリストから削除され、検索した用語は置き換えられません。

1. リスト上部の右側にある&#x200B;**すべて置換** \（![](images/replace-all-in-file-icon.svg)\）をクリックすると、検索したすべてのファイル内の検索語のすべての出現箇所を、ワンクリックで置換語に置き換えることができます。

   >[!NOTE]
   >
   > **すべてを置き換え** アイコンを有効にするには、システム管理者は、**エディター設定**&#x200B;の「**一般**」タブで「**すべてを置き換え**」オプションを選択する必要があります。


すべての操作を一度に1つだけ置換できます。この操作はシステム全体で一度に実行できます。この操作が実行されるまでは、「すべての処理中を置換」ステータスが表示されます。 間にあるすべての置換の操作を中止するか、ログレポートを表示することもできます。 操作を中止すると、その操作に関する通知がインボックスに届きます。 選択したファイルのすべてのオカレンスを置き換えると、成功通知が表示されます。

![](images/replace-all-in-progress.png){width="400"}

マップの&#x200B;**オプション** メニューの「**マップ内で検索**」オプションを使用して、マップ内のテキストを検索および置換することもできます。 このオプションは、リポジトリパネルまたはマップビューで開いたマップに対して表示されます。

## コンテンツ編集エリア {#id2051EB000UI}

コンテンツ編集エリアは、トピックまたはマップのコンテンツが表示される場所です。 この領域ですべてのコンテンツを編集します。 WYSIWYGで編集中のコンテンツが表示されます。 複数のトピックを同時に開くことができ、それぞれのタブに表示されます。

デフォルトでは、タブでファイルタイトルを表示できます。 ファイルにカーソルを合わせると、ファイルタイトルとファイルパスがツールチップとして表示されます。

>[!NOTE]
>
> 管理者は、タブ内のファイル名でファイルのリストを表示することもできます。 **ユーザー設定** ![](images/user_preference_editor_icon.svg)の「**でファイルを表示」セクションの「** ファイル名&#x200B;**」オプションを選択します。**

ファイルのタブの下に、現在のカーソル位置にエレメントのパンくずリストがあります。 コンテンツ編集領域の右上隅に、現在のトピックのバージョン番号が表示されます。

![](images/content-editing-area.png){width="650"}

## 右側のパネル {#id2051EB003YK}

右側のパネルは、現在選択されているドキュメントに関する情報を含む永続的なパネルです。

>[!NOTE]
>
> 右側のパネルはサイズ変更可能です。 パネルのサイズを変更するには、パネルの境界にカーソルを置き、カーソルを両方向矢印に変え、クリックしてドラッグしてパネルの幅のサイズを変更します。

右側のパネルでは、次の機能にアクセスできます。

**コンテンツのプロパティ** - ![&#x200B; コンテンツのプロパティ &#x200B;](images/content-properties-icon.svg)

右側のパネルで「**コンテンツのプロパティ**」アイコンを選択すると、**コンテンツのプロパティ**&#x200B;機能にアクセスできます。 **コンテンツのプロパティ** パネルには、ドキュメント内で現在選択されている要素のタイプとその属性に関する情報が含まれています。

**タイプ**：現在のタグの完全な階層のタグを表示して、ドロップダウンから選択できます。

**属性**: **属性** ドロップダウンパネルは、レイアウトビュー、作成者ビュー、Source ビューで使用できます。 属性を簡単に追加、編集、削除できます。

1. 「**+ Add**」をクリックします。

   ![&#x200B; コンテンツプロパティの属性](images/properties-tab-attributes_cs.png){width="300"}

1. **属性** ドロップダウンパネルで、ドロップダウンリストから属性を選択し、属性の値を指定します。  次に、**追加**&#x200B;をクリックします。

   複数の属性![&#128279;](images/attributes-multiple-properties.png){width="300"}を持つ属性パネル

1. 属性を編集するには、属性にカーソルを合わせ、**編集** ![編集アイコン &#x200B;](images/edit_pencil_icon.svg)を選択します。
   ![属性を編集](images/edit-attributes-content-properties.png){width="300"}

1. 属性を削除するには、属性にカーソルを合わせ、**削除** ![削除アイコン &#x200B;](images/Delete_icon.svg)を選択します。


>[!NOTE]
>
> トピックに参照コンテンツが含まれている場合でも、プロパティパネルを使用してトピックに属性を追加できます。

管理者が属性のプロファイルを作成した場合は、これらの属性と設定値を取得します。 コンテンツプロパティパネルを使用して、これらの属性を選択し、トピック内の関連コンテンツに割り当てることができます。 この方法で条件付きコンテンツを作成し、条件付き出力を作成するために使用することもできます。 コンディショナルプリセットを使用した出力の生成について詳しくは、[&#x200B; コンディショナルプリセットの使用](generate-output-use-condition-presets.md#)を参照してください。



**ファイルのプロパティ** - ![](images/topic-properties-icon.svg)

右側のパネルの「ファイルのプロパティ ![](images/topic-properties-icon.svg)」アイコンをクリックして、選択したファイルのプロパティを表示します。 ファイルプロパティ機能は、レイアウト、オーサー、Source、プレビューの4つのモードまたはビューで使用できます。

ファイルのプロパティには、次の2つのセクションがあります。

**一般**

「一般」セクションでは、次の機能にアクセスできます。

![file-properties](images/file-properties-general.png){width="300"}

- **名前**：選択したトピックのファイル名を表示します。 ファイル名は、選択したファイルのプロパティページにハイパーリンクされます。
- **ID**：選択したトピックのIDを表示します。
- **タグ**：トピックのメタデータタグです。 これらは、プロパティページの「タグ」フィールドから設定します。 ドロップダウンから入力または選択できます。  タグはドロップダウンの下に表示されます。 タグを削除するには、タグの横にある十字アイコンを選択します。
- **他のプロパティを編集**: ファイルのプロパティ ページから、他のプロパティを編集できます。
- **言語**: トピックの言語を表示します。 プロパティページの「言語」フィールドから設定します。
- **作成日**: トピックが作成された日時を表示します。
- **チェックアウト者**: トピックをチェックアウトしたユーザーを表示します。
- **ドキュメントの状態**：現在開いているトピックのドキュメントの状態を選択して更新できます。 詳しくは、[&#x200B; ドキュメントの状態&#x200B;](web-editor-document-states.md#)*を参照してください。*

**メモ：** ファイル プロパティの様々なフィールドの属性値をクリップボードにコピーできます。

**参照**

「参照」セクションでは、次の機能にアクセスできます。

![](images/file-properties-references.png){width="300"}

- **使用済み**：使用済み参照は、現在のファイルが参照または使用されているドキュメントを一覧表示します。
- **送信リンク：**&#x200B;送信リンクには、現在のドキュメントで参照されているドキュメントが一覧表示されます。

デフォルトでは、タイトルでファイルを表示できます。 ファイルにカーソルを合わせると、ファイルタイトルとファイルパスがツールチップとして表示されます。

>[!NOTE]
>
> 管理者は、Web エディターでファイル名でファイルのリストを表示することもできます。 **ユーザー設定** ![](images/user_preference_editor_icon.svg)の「**でファイルを表示」セクションの「** ファイル名&#x200B;**」オプションを選択します。**

**メモ：**&#x200B;すべての使用済み参照と出力参照は、文書にハイパーリンクされています。 リンクされたドキュメントを簡単に開いて編集できます。

ファイルを開くだけでなく、「参照」セクションの「**オプション**」メニューを使用して、多くのアクションを実行することもできます。 実行できるアクションには、編集、プレビュー、UUIDのコピー、パスのコピー、お気に入りに追加、プロパティ、マップを開くダッシュボードなどがあります。

**レビュー** - ![](images/review-icon.svg)

レビューアイコンをクリックすると、レビューパネルが開き、現在開いているドキュメントのレビュータスクを作成できます。

![](images/review-panel-before-opening.png){width="300"}

複数のレビュープロジェクトを作成した場合は、ドロップダウンから1つを選択して、レビューコメントにアクセスできます。

レビューパネルを使用すると、トピックに対するコメントの返信を表示および投稿できます。 コメントは1つずつ承認または却下できます。

詳細については、[&#x200B; アドレスのレビューのコメント &#x200B;](review-address-review-comments.md#)を参照してください。

**変更履歴** - ![](images/track-change-icon.svg)

右側のパネルの「変更履歴」機能を使用すると、ドキュメントで行われたすべての更新の情報を表示できます。 ドキュメントに加えられた特定の更新を検索することもできます。

>[!NOTE]
>
> 変更履歴は、メインツールバーの「変更履歴の有効化/無効化」機能を使用して追跡されたすべての更新を表示します。 詳しくは、[変更のトラックの有効化/無効化](#id205DF0203Y4)を参照してください。
