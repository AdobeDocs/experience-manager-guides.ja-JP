# トピックの作成 {#id2056AL00O5Z}

AEM Guidesでは、トピック、タスク、コンセプト、参照、用語集、DITAVALなどのDITA トピックを作成できます。 すぐに使えるテンプレートを利用してトピックを作成するだけでなく、独自のテンプレートを定義することも可能です。 テンプレート選択ブループリントとWeb エディターに表示するには、これらのテンプレートをフォルダープロファイルに追加する必要があります。

グローバルプロファイルとフォルダープロファイルの設定は、フォルダーレベルの管理ユーザーのみが使用できます。 グローバルおよびフォルダーレベルのプロファイルの設定について詳しくは、「Adobe Experience Manager Guidesのインストールと設定」の「*オーサリングテンプレートの設定*」を参照してください。

トピックを作成するには、次の手順を実行します。

1. Assets UIで、トピックを作成する場所に移動します。

1. 新しいトピックを作成するには、**作成** \> **DITA トピック**&#x200B;をクリックします。

1. ブループリントページで、作成するDITA ドキュメントのタイプを選択し、**次へ**&#x200B;をクリックします。

   ![](images/create_dita_topic.png){width="800"}

   デフォルトでは、AEM Guidesは最も一般的に使用されるDITA トピックテンプレートを提供します。 組織の要件に応じて、さらにトピックテンプレートを設定できます。「*オーサリングテンプレートの設定*」を参照して、設定に合わせてAdobe Experience Manager Guidesをインストールおよび設定してください。

   >[!NOTE]
   >
   > Assets UIのリストビューでは、DITA トピックタイプは、「タイプ」列に「トピック」、「タスク」、「コンセプト」、「参照」、「Glossentry」、または「DITAVAL」として表示されます。 DITA マップはマップとして表示されます。

1. プロパティ ページで、文書&#x200B;**タイトル**&#x200B;を指定します。

1. \（Optional\） ファイル **Name**&#x200B;を指定します。

   管理者がUUID設定に基づいて自動ファイル名を設定している場合、ファイル名を指定するオプションは表示されません。 UUID ベースのファイル名が自動的にファイルに割り当てられます。

   ファイルの命名オプションが使用可能な場合は、ドキュメントの&#x200B;**タイトル**&#x200B;に基づいて名前も自動的に提案されます。 文書名を手動で指定する場合は、**Name**&#x200B;にスペース、アポストロフィ、または中括弧が含まれておらず、.xmlまたは.ditaで終わっていることを確認してください。 デフォルトでは、AEM Guidesはすべての特殊文字をハイフンに置き換えます。 DITA ファイルの命名に関するベストプラクティスについては、ベストプラクティスガイドの「ファイル名」セクションを参照してください。

1. 「**作成**」をクリックします。 トピック作成メッセージが表示されます。

   Web エディターで編集するトピックを開くか、AEM リポジトリにトピックファイルを保存するかを選択できます。

   Assets UI **Create** \> **DITA トピック**&#x200B;またはWeb エディターから作成したすべての新しいトピックには、一意のトピック IDが割り当てられます。 このIDの値は、ファイル名そのものです。 また、新しいドキュメントは、DAMでトピックの最新の作業コピーとして保存されます。 新しく作成したトピックのリビジョンを保存するまで、バージョン履歴にバージョン番号は表示されません。 編集のためにトピックを開くと、トピックファイルのタブの右上隅にバージョン情報が表示されます。

   ![](images/topic-version-none_cs.png){width="550"}

   新しく作成されたトピックのバージョン情報は&#x200B;*none*&#x200B;として表示されます。 新しいバージョンを保存すると、バージョン番号が1.0として割り当てられます。 新しいバージョンの保存について詳しくは、[新しいバージョンとして保存](web-editor-features.md#save-as-new-version-id209ME400GXA)を参照してください。


>[!NOTE]
>
> 管理者が編集前にファイルをチェックアウトするようにWeb エディターを設定している場合、チェックアウトするまでファイルを編集できません。 同様に、設定されている場合は、チェックアウトしたファイルを閉じる前にチェックインするように求められます。

>[!IMPORTANT]
>
> DITA トピックを作成したら、作業コピーへの変更を保存し続け、トピックの更新が完了したら新しいバージョンを作成します。
